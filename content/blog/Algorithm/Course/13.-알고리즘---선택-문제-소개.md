# 1. 선택 문제

이번에는 선택(selection) 문제라고 하는 몇 가지 새로운 문제를 살펴볼 것이다.

## 1-1. 입력과 출력 (+ 목표)

```
입력 : n개의 값과 k(1 <= k <= n) 값
출력 : k번째로 작은 입력 값
목표 : 비교 횟수 최소화
```

- 입력의 크기 n에 대해, 'n개의 값' 과 '1과 n 사이에 있는 k라는 값' 이 입력으로 주어진다.
- 이러한 입력에 대해, n개의 값 중에서 k번째로 작은 입력 값을 찾아서 출력해야 한다.
   - 예를 들어 k가 3이라면, n개의 값 중에서 3번째로 작은 값을 찾아야 한다.  
     `(마찬가지로, k가 10이면, 10번째로 작은 값을 찾으면 된다.)`
- k번째로 작은 값을 구하기 위해서는, 서로 다른 두 개의 수를 계속 비교해야 한다.
   - 이 때, 최대한 적은 비교 횟수를 이용해 문제를 해결하는 것이 중요하다.
   - 따라서, 이렇게 비교 횟수를 되도록 작게 하는 것이 선택 문제의 목표라고 할 수 있다.

## 1-2. 최대값 찾기 문제 (k = n)

k = n 이라는 것은, n개의 숫자 중에서 n번째로 작은 숫자를 구해야 한다는 것을 의미한다.

> 이는 최대값이므로, 결국, 이것은 최대값을 찾는 문제와 똑같다고 할 수 있다.

<br>

최대값을 찾는 문제는 이전 강의에서 'arrayMax' 라는 알고리즘과 함께 살펴봤다.

```
A = [o, o, o, o, o, o, o, o, o]
     ↑  ^  ^  ^  ^  ^  ^  ^  ^  <- (n - 1) 번 비교
    c.M
```

- n개의 숫자가 담긴 A라는 리스트가 입력으로 주어진다고 가정한다.
- 맨 처음에는 A[0] 를 currentMax 로 선택하고, 값을 하나씩 확인하면서 비교해나간다.
- 현재 보고 있는 A[i] 의 값이 currentMax 보다 크면, currentMax 는 A[i] 가 된다.
- 이 때, 처음에 currentMax 로 지정되는 A[0] 을 제외한 (n - 1) 개를 비교하게 된다.
- 즉, '어떤 경우라도 총 (n - 1) 번의 비교를 하면 최대값을 찾을 수 있다' 라고 할 수 있다.
- 또한, (n - 1) 보다 더 적은 횟수의 비교를 통해 최대값을 찾는 방법은 없다.  
  `(이는 최소한 (n - 1) 번의 비교를 수행해야 함을 의미한다.)`

## 1-3. 상한과 하한

```
n - 1 번의 비교 : 최대값 찾기 가능 -> 상한(upper bound)
n - 1 번의 비교가 필요한 경우 존재 -> 하한(lower bound)

상한 = n - 1 = 하한 <- 완전히 풀림
```

위에서 살펴봤듯, (n - 1) 번의 비교를 수행하면, 어떤 경우에 대해서도 최대값을 찾을 수 있다.

- 이는, (n - 1) 번 비교하는 알고리즘은 최대값을 무조건 찾을 수 있다는 것을 의미한다.
- 이렇게, 어떤 경우라도 문제 해결이 가능한 연산 횟수를 '상한(upper bound)' 이라고 한다.

<br>

최대값을 찾기 위해, 반드시 (n - 1) 번의 비교를 수행해야 하는 경우가 있다고 가정해보자.

- 이는, 어떤 좋은 알고리즘을 사용하더라도, 최소한 (n - 1) 번 비교해야 함을 의미한다.
- 이런 상황에서, 문제 해결에 필요한 최소한의 연산 횟수를 '하한(lower bound)' 이라고 한다.

<br>

최대값 찾기 문제에서 상한은 '항상 최대값을 찾을 수 있는 비교 횟수' 인 (n - 1) 이다.

- 이 때, 최대값을 찾기 위해 상한 횟수인 (n - 1) 보다 많이 비교할 필요는 없다.
- 상한 이하의 비교만으로도 항상 최대값을 찾는 arrayMax 알고리즘이 있기 때문이다.
- 따라서, '어떤 문제를 풀기 위해서 상한 이상의 연산을 수행할 필요는 없다' 고 할 수 있다.

<br>

최대값 찾기 문제에서 하한은 '최대값을 찾기 위한 최소한의 비교 횟수' 인 (n - 1) 이다.

- 어떤 알고리즘에 대해서도, 하한 횟수 이상의 연산을 수행해야 하는 경우가 존재한다.
- 따로 증명하지는 않았지만, (n - 1) 번 이상의 비교가 필요한 경우는 이미 증명되어 있다.

## 1-4. 최대값 찾기 문제 정리

결국, 최대값 찾기 문제의 상한과 하한은 같다고 할 수 있다.

- 상한은 해당 횟수가 어떤 문제에서도 충분하다는 것을 의미한다.
- 하한은 해당 횟수가 어떤 경우에는 필요하다는 것을 의미한다.

<br>

따라서, 상한과 하한이 일치한다는 것의 의미는 아래와 같이 정리된다.

- 필요한 만큼의 비교 횟수만으로 항상 원하는 값을 찾을 수 있다.
- 따라서, 최대값을 찾으려면 반드시 (n - 1) 의 비교가 필요하다.
- 같은 이유로, (n - 1) 번의 비교는 항상 최대값을 찾기에 충분하다.

<br>

상한과 하한이 일치하기 때문에, 이 문제는 완전히 풀린 것이라고 할 수 있다.

- 왜냐하면, (n - 1) 번의 비교가 필요한 경우가 반드시 있고,  
  (n - 1) 번의 비교로 항상 최대값을 찾을 수 있기 때문이다.
- 이는 풀리지 않은 문제가 없다는 것을 의미하므로, 완전하게 풀린 것이라고 할 수 있다.

# 2. 토너먼트 선택 (k = n)

이렇게 왼쪽에 있는 값부터 하나씩, (n - 1) 번 비교해서 최대값을 찾을 수 있다.

- 하지만, 반드시 이 방법을 사용해야 하는 것은 아니며, 다른 방법도 있다.
- 바로, 토너먼트 방식으로 진행하는 것(Tournament Selection) 이다.

## 2-1. 예시

```
A = [○, ○, ○, ○, ○, ○, ○, ○, ○]
      \/    \/    \/    \/   |
       ○    ○      ○    ○    ○
        \__/        \__/     |
          ○          ○       ○
           \________/        |
                ○            ○
                 \__________/
                      ○ <- 최대값
```

예를 들어, A[0] 와 A[1] 을 비교하면, 그중에서도 큰 값이 있다.

- 이렇게 2개의 원소를 짝지어 비교하고, 둘 중에 더 큰 값을 찾아 선택한다.
- 끝까지 반복하고 마지막에 하나가 남는다면, 그 원소는 부전승으로 취급한다.

<br>

이렇게 선택된 값들을 다시 2개씩 묶어서 다음 회차(round) 를 진행하면 된다.

- 여기서도 마찬가지로, 둘 중에 더 큰 값이 그다음 회차에 진출하게 된다.
- 이러한 과정이 반복되면서, 다음 회차의 값들도 그다음 회차로 진출한다.
- 마지막으로 만나는 둘 중에 더 큰 값이 우승하며, 그 값이 최대값이 된다.

## 2-2. 정리

```
○   ○   ○   ○   ○   ○   ○   ○   ○
 \_/     \_/     \_/     \_/    |  <- 1
  ○       ○       ○       ○     ○
   \_____/         \_____/      |  <- 2
      ○               ○         ○
       \_____________/          |  <- 3
              ○                 ○
               \_______________/   <- 4
                       ○

1회차 : (\/ * 4) => 4             = 1
2회차 : (\/ * 2) => 4 + 2         = 6
3회차 : (\/ * 1) => 4 + 2 + 1     = 7
4회차 : (\/ * 1) => 4 + 2 + 1 + 1 = 8
```

위의 예시에서는 총 9개의 숫자가 주어졌는데, 이 때의 비교 횟수를 확인해보자.

- 1회차에서는 (비교 4번 + 부전승) 으로, 총 4번의 비교가 수행되었다.
- 2회차에서는 (비교 2번 + 부전승) 으로, 총 6번의 비교가 수행되었다.
- 3회차에서는 (비교 1번 + 부전승) 으로, 총 7번의 비교가 수행되었다.
- 마지막 회차(결승) 에서 수행된 비교까지 총 8번의 비교가 수행되었다.

<br>

이 때, n = 9 이며, 최대값을 찾기 위해서 (n - 1) 번인 8번을 비교한 것이다.

- 이렇게 토너먼트 방식도 마찬가지로, (n - 1) 번의 비교를 통해 최대값을 찾을 수 있다.
- 왼쪽부터 차례대로 찾든, 토너먼트 방식으로 비교해서 찾든, (n - 1) 번의 비교가 필요하다.
- 따라서, '(n - 1) 번의 비교를 통해 최대값을 찾는 방법은 항상 존재한다' 고 할 수 있다.

<br>

# 3. 선택 문제 (k = 1, n)

## 3-1. 최대값 찾기와 최소값 찾기

지금까지의 내용에 대한 결론을 간단히 정리하자면 아래와 같다.

- 선택 문제에서 k = n 인 경우는, 최대값을 찾는 경우이다.
- 최대값을 찾기 위해 (n - 1) 번의 비교가 필요한 경우가 있다.
- (n - 1) 번의 비교를 수행하면 항상 최대값을 찾을 수 있다.

<br>

다음으로 살펴볼 것은 k = 1 인 경우, 즉, 1번째로 작은 값을 찾는 문제다.

- 이것은 최소값을 찾는 문제이기 때문에, 최대값 찾기 방법을 반대로 적용하면 된다.
- 따라서, 최대값과 마찬가지로, (n - 1) 번의 비교를 통해 항상 최소값을 찾을 수 있다.

## 3-2. 최대값, 최소값 찾기

이번에는 k = 1, n 인 경우, 즉, 최대값과 최소값을 동시에 찾는 문제를 살펴보자.

```
A = [○, ○, ○, ○, ○, ○, ●, ○] -> (n - 1) 번 비교 : 최대값 (●)

A = [○, ○, ○, ○, ○, ○, ○] ●  -> (n - 2) 번 비교 : 최소값

최대값, 최소값을 구하기 위해 필요한 비교 횟수 : (n - 1) + (n - 2) = 2n - 3
```

8개의 숫자가 주어졌다고 가정했을 때, 아래와 같은 방법으로 문제를 풀 수 있다.

- 우선, 배열의 모든 원소를 차례대로 (n - 1) 번 비교하여 최대값을 찾는다.
- 다음으로, 최대값을 제외한 나머지 (n - 1) 개의 원소 중에서 최소값을 찾는다.
   - 이 때, 원소의 개수가 (n - 1) 개이므로, 비교 횟수는 (n - 1) - 1 = (n - 2) 다.
- 이렇게 총 (n - 1) + (n - 2) = (2n - 3) 번의 비교로 최대값과 최소값을 찾을 수 있다.

<br>

어떤 경우에도 (2n - 3) 번의 비교로 최대값, 최소값을 동시에 찾을 수 있다는 것을 확인했다.

- 따라서, 최대값, 최소값 찾기 문제의 상한은 (2n - 3) 이라 할 수 있다.
- 그런데, 선택 문제의 목표는 비교 횟수를 최대한 적게 만드는 것이다.
- 그러므로, 더 적은 비교 횟수로 문제를 해결하는 방법을 고민해 볼 필요가 있다.  
  `(이는 상한을 더 낮추는 방법이라고도 할 수 있다.)`

## 3-3. 토너먼트 선택

다시 8개의 숫자가 있다고 가정하고, 토너먼트 방식을 이용해 문제를 풀어보자.

### 3-3-1. 최대값 찾기

토너먼트 방식도 최대값을 찾기 위해, 총 (n - 1) 번의 비교가 필요하다.

```
[○, ○, ○, ○, ○, ○, ○, ○]
  \/    \/    \/    \/    (\/ * 4) => 4         = 4
   ○    ○      ○    ○
    \__/        \__/      (\/ * 2) => 4 + 2     = 6
      ○          ○
       \________/         (\/ * 1) => 4 + 2 + 1 = 7 = n - 1 (n = 8)
           ○ <- 최대값
```

- 2개의 수를 비교하면, 항상 둘 중 하나가 더 크다.
- 비교를 반복하여 마지막에 남은 값이 최대값이다.
- n = 8 일 때의 (n - 1), 총 7번의 비교가 수행된다.

### 3-3-2. 최소값 찾기

최대값 찾기 1회차에서 비교한 두 개의 값 중, 더 큰 값은 다음 회차로 진출했다.

```
큰 값 : ●, 작은 값 : ◎

[◎, ●, ●, ◎, ●, ◎, ◎, ●]
```

- 이 때, 2회차에 진출하지 못한 4개의 값들(◎) 중에 반드시 최소값이 존재한다.
- 왜냐하면, 가장 큰 값이 우승하는 토너먼트에서 가장 먼저 탈락했기 때문이다.

<br>

이 때, 1회차에서 탈락한 값들은 총 4개, 즉 (n / 2) 개다.

- n 이 짝수라면 나누어떨어질 것이고, 홀수라면 하나가 남을 것이다.
- 이렇게 탈락한 값들, 즉, (n / 2) 개의 값 중에 최소값이 있는 것이다.

<br>

따라서, 최대값을 뺀 나머지 (n - 1) 개에서 최소값을 찾지 않아도 된다.

- 대신, 이러한 (n / 2) 개의 값 중에서 최소값을 찾으면 된다.
- 이 때, 최소값은 (n / 2) - 1 번의 비교를 통해 찾아낼 수 있다.

### 3-3-2. 비교 횟수 정리

따라서, 최대값은 (n - 1) 번, 최소값은 (n / 2) - 1 번의 비교를 통해 찾을 수 있다.

- 두 비교 횟수를 합치면, (n - 1) + (n / 2) - 1 = ((3 / 2) * n) - 2 가 된다.
- ((3 / 2) * n) - 2 번의 비교를 통해 최대값과 최소값을 찾을 수 있게 된 것이다.

## 3-4. 상한과 하한

앞에서 살펴본 방법은 문제 해결을 위해 (2n - 3) 번의 비교를 수행해야 했다.

- 하지만, 토너먼트 방식을 이용해 비교 횟수를 ((3 / 2) * n) - 2 로 줄일 수 있었다.
   - 이는, 더 적은 비교 횟수로도 문제를 풀 수 있다는 것(상한) 을 증명한 것이다.
- 따라서, 최대값과 최소값을 동시에 찾는 문제의 상한은 ((3 / 2) * n) - 2 라고 할 수 있다.

<br>

이 때, 하한은 문제를 해결하는 데 필요한 최소한의 비교 횟수다.

- 위에서 살펴봤듯, ((3 / 2) * n) - 2 번의 비교가 필요한 경우가 반드시 있다.
- 따라서, 최대값과 최소값을 동시에 찾는 문제의 하한도 증명되었다고 할 수 있다.

<br>

결국, 최대값과 최소값을 동시에 찾는 문제의 상한과 하한이 같다고 할 수 있다.

- ((3 / 2) * n) - 2 번의 비교가 필요한 경우가 있다.
- ((3 / 2) * n) - 2 번의 비교로 문제를 풀 수 있다.

<br>

> 이번 수업에서는 이 문제의 하한에 대해 따로 증명하지 않을 것이다.

# 4. 선택 문제 (k = 1, 2)

이번에는 k = 1, 2 인 경우, 즉, 가장 작은 값과 두 번째로 작은 값을 찾는 문제를 살펴보자.

## 4-1. 문제 파악

이 문제를 해결하는 방법 중, 가장 쉬운 방법은 아래와 같다.

- 우선, 배열의 모든 원소를 차례대로 (n - 1) 번 비교하여 최소값을 찾는다.
- 그리고, 최소값을 제외한 나머지 (n - 1) 개의 원소 중에서 다시 최소값을 찾는다.
   - 이는 전체에서 두 번째로 작은 값이며, 비교 횟수는 (n - 1) - 1 = (n - 2) 다.
- 이렇게 총 (n - 1) + (n - 2) = (2n - 3) 번의 비교로 원하는 값들을 찾을 수 있다.

<br>

(2n - 3) 보다 더 적은 비교 횟수로 문제를 해결할 수 있는지에 대해 알아보자.

- 앞에서 살펴본 최대값과 최소값을 동시에 찾는 방법처럼, 토너먼트 방식을 이용할 수 있다.
   - 이 때, 최대값을 찾기 위해 수행했던 비교의 결과들을 최소값을 찾는 데 이용했다.
- 이번 문제에서도 마찬가지로, 최소값을 찾기 위해서 반드시 (n - 1) 번은 비교해야 한다.
   - 또, 이러한 비교 과정에서 얻은 정보를 이용해 두 번째로 작은 값을 찾을 수 있다.
- 이렇게, 이미 수행된 비교를 통해 더 유용한 정보를 얻어서 전체 비교 횟수를 줄이면 된다.

## 4-2. 토너먼트 선택

```
[○, ○, ○, ○, ○, ○, ○, ○]
  \/    \/    \/    \/
   ○    ○      ○    ○
    \__/        \__/
      ○          ○
       \________/
           ○ <- 최소값
```

- 이전처럼 둘씩 비교하지만, 이번엔 더 작은 값이 다음 회차에 진출하게 된다.
- 이러한 과정이 반복되면서, 다음 회차의 값들도 그다음 회차로 진출한다.
- 마지막으로 만나는 둘 중에 더 작은 값이 우승하며, 그 값이 최소값이 된다.

### 4-2-1. 예시1

```
[○, ○, ◎, ○, ○, ●, ○, ○]
  \/    \/    \/    \/
   ○    ◎      ●    ○
    \__/        \__/
      ◎          ●
       \________/
           ● <- 최소값
```

이 때, 6번째 위치의 값(●) 이 가장 작은 값이라고 가정해보자.

- 위와 같은 흐름으로 결승까지 작은 값으로 선택된다.
- 결승에서도 작은 값으로 선택되어 결국 우승하게 된다.

<br>

3번째 위치의 값(◎) 이 두 번째로 작은 값이라고 가정해보자.

- 두 번째로 작은 값이기 때문에, 결승까지는 무조건 작은 값으로 선택된다.
- 하지만, 토너먼트 과정 중에, 언젠가는 가장 작은 값과 만날 수밖에 없다.

### 4-2-2. 예시2

두 번째로 작은 값(◎) 과 가장 작은 값(●) 은 결승이 아닌 시점에 비교될 수도 있다.

```
[○, ○, ○, ○, ◎, ●, ○, ○]
  \/    \/    \/    \/
   ○    ○      ●    ○
    \__/        \__/
      ○          ●
       \________/
           ● <- 최소값
```

- 위와 같이, 첫 번째 회차에서 가장 작은 값과 두 번째로 작은 값이 만날 수도 있다.
- 이러한 상황에서, 두 번째로 작은 값은 결승까지 가지 못하고 바로 탈락하게 된다.

<br>

따라서, "두 번째로 작은 값은 '가장 작은 값과 비교되어 탈락한 값' 중에 있다" 고 할 수 있다.

```
[○, ○, ○, ○, ◎, ●, ○, ○]
  \/    \/    \/    \/
   ○    ○      ●    ◎
    \__/        \__/
      ◎          ●
       \________/
           ● <- 최소값
```

- 최소값(●) 과 비교된 값들(◎) 중에서 두 번째로 작은 값이 있는 것이다.
- 따라서, 그러한 값 중에서 어떤 값이 가장 작은 값인지를 비교해보면 된다.
   - 그리고, 이 중에서 가장 작은 값이 전체에서 두 번째로 작은 값이 된다.

### 4-2-3. 비교 횟수 정리

```
(n - 1)     : 최소값을 위한 비교 횟수
(round - 1) : 두 번째 최소값을 위한 비교 횟수
```

- 맨 처음에, 최소값을 찾는 데 필요한 비교 횟수는 (n - 1) 번이다.
- 그리고, 최소값과 비교된 값의 수는 진행된 회차의 수인 3이 된다.
- 따라서, (회차 - 1) 번을 비교하면, 두 번째로 작은 값을 찾을 수 있다.

## 4-3. 토너먼트 회차 수

```
[○, ○, ○, ○, ○, ○, ○, ○, ○]
  \/    \/    \/    \/   |
   ○    ○      ○    ○    ○
    \__/        \__/     |
      ○          ○       ○
       \________/        |
           ○             ○
            \___________/
                  ○
```

위에서 살펴봤듯, n = 8 일 때, 진행해야 하는 회차 수는 3이다.

- n = 9 일 때, 부전승하는 항목이 생기면서, 1번의 회차가 추가된다.
   - 따라서, n = 9 일 때 진행되는 회차 수는 3 + 1 = 4 다.
- n = 10 이 되어도 진행해야 하는 회차 수는 여전히 4다.
   - 이후에, n = 16 이 되기 전까지는 계속 4회차로 같다.
- 진행해야 하는 회차 수는 n = 17 ~ 32 까지 5, n = 33 부터 6이다.

<br>

이렇게 n개의 숫자가 있을 때, 회차 수를 구하기 위해선 log2(n) 을 구해야 한다. 

- log2(n) = 3 (n = 8), log2(n) = 4 (n = 9) 이므로, log2(n) 에 올림을 해야 한다.
- 그러면, ${\lceil}log2(8){\rceil}$ 은 정확히 3, ${\lceil}log2(9){\rceil}$ 은 3.xx 를 올림 하여 4가 된다.

## 4-4. 상한과 하한

마지막으로, 최소값을 위한 비교 횟수와 두 번째로 작은 값을 위한 비교 횟수를 합해야 한다.

```
(n - 1) + ((log2(n) 의 올림) - 1) = n - (log2(n) 의 올림) - 2
```

- 총, $(n - 1) + ({\lceil}log2(n){\rceil} - 1) = n - {\lceil}log2(n){\rceil} - 2$ 의 비교를 수행해야 한다.
- 즉, $n - {\lceil}log2(n){\rceil} - 2$ 번 비교하면, 최소값과 두 번째로 작은 값을 항상 찾을 수 있다.
- 따라서, 최소값과 두 번째로 작은 값을 구하는 문제의 상한은 $n - {\lceil}log2(n){\rceil} - 2$ 다.
- 그리고, $n - {\lceil}log2(n){\rceil} - 2$ 번 비교해야 하는 경우가 있다는 것(하한) 도 위에서 증명되었다.

<br>

이렇게 상한과 하한이 일치하기 때문에, 이 문제는 완전히 풀린 것이라고 할 수 있다.

> 마찬가지로, 최대값과 두 번째로 큰 값을 찾는 문제도 같은 방식을 반대로 하면 된다.

<br>

이번 수업에서는, (1 / n / 1, n / 1, 2) 처럼 특별한 k 값에 대한 선택 문제들을 살펴봤다.

다음 수업에서는, 일반적인 k에 대해, k번째로 작은 값을 찾는 방법에 대해 살펴볼 것이다.

> 여기서 '일반적인' 의 의미는 k가 '입력으로 주어지는 임의의 값' 일 때를 말한다.
