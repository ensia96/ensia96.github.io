# 1. 큰 수 곱셈 문제

이번 수업에서는, '두 개의 큰 수를 얼마나 효율적으로 곱셈할 수 있는가?' 에 대한 문제를 살펴보자.

- 이러한 문제는, 분할 정복법을 사용하는, 비교적 유명한 알고리즘을 활용하여 해결할 수 있다.
- 이는, 제안한 사람의 이름을 따서, '카라추바 알고리즘(Karatsuba Algorithm)' 이라고 한다.

## 1-1. 문제 설명

프로그래밍 언어, 특히, C 언어에서는 정수와 실수를 나타낼 수 있는 여러 가지 타입을 제공한다.

- 이 때, 실수를 표현하는 타입은 '부동 소수점 타입(floating point type)' 이라고 부른다.
- 정수는 short, int, long, longlong, 실수는 float, double, longdouble 등이 제공된다.
- 사실, 이러한 숫자 타입들은 최소 2바이트, 최대 32바이트에 해당하는 메모리를 사용한다.
- 이와 같이, 바이트 수에 제한이 있어서, 큰 수나 정밀한 실수를 표현하는 데에 한계가 있다.

<br>

> 이러한 '많은 자리수를 갖는 수' 를 표현하는 방법에 대해서는, 이후에 다른 수업에서 살펴볼 것이다.

<br>

이전의 수업에서 살펴봤듯, RAM 모델은, 상수 시간, 즉, 단위 시간 내에 두 수의 곱셈을 수행한다.

- 숫자의 크기/정밀도에 상관없이, 단위 시간 내에 곱셈을 수행하는 컴퓨터 모델이라고 가정했다.
- 하지만, 이것은 가상의, 이상적인 컴퓨터 모델이었기 때문에 가능했던 것이지, 현실은 다르다.
- 물론, 정수형(int) 타입의 두 숫자에 대한 곱셈은, 현실에서도, 상수 시간 내에 수행될 수 있다.
- 하지만, 자리수가 많은 두 수를 곱하려면, 상수 시간보다 훨씬 더 많은 시간이 필요할 것이다.

<br>

> 이 때, '그러한 두 숫자를 더 효율적으로, 더 빠르게 곱하는 방법을 찾는 문제' 가 '큰 수 곱셈' 문제다.

## 1-2. 기본적인 곱셈법

임의의 n자리 숫자 A, B가 입력으로 주어진다고 가정하고, 기본적인 곱셈 과정에 대해 살펴보자.

```
A = a(n - 1) a(n - 2) ... a1 a0
B = b(n - 1) b(n - 2) ... b1 b0
```

- A의 맨 왼쪽 숫자를 a(n - 1) 이라고 하면, a0은 1의 자리, a1은 10의 자리라고 할 수 있다.
- 그러므로, A라는 숫자의 각 자리수를, a(n - 1) a(n - 2) .. a1 a0 의 형태로 표현할 수 있다.
- 굳이 n자리가 아니어도 상관없지만, n자리 숫자라고 가정했던 B의 경우도, 이와 마찬가지다.

<br>

이러한 두 수를 곱한다고 가정했을 때, 실제 학교에서 배우는 기본적인 곱셈의 과정은 다음과 같다.

```
                              c1
        a(n - 1) a(n - 2) ... a1 a0 = A
    *   b(n - 1) b(n - 2) ... b1 b0 = B
    ───────────────────────────────
      []   []       [] ... [] [] []  <-  b0 * (a(n - 1) ... a0)
   [] []   []       [] ... [] []  0  <-  b1 * (a(n - 1) ... a0)

                   ፧

[] [] []    0        0 ...  0  0  0  <-  b(n - 1) * (a(n - 1) ... a0)
            └──────────┬──────────┘
                    (n - 1)
───────────────────────────────────
[]    ...    ...    ...    [] [] []  <-  A * B
```

- b0 * a0 의 값을 아래에 적고, 올림수(carry) 가 있으면, 그것(c1) 을 다음 자리로 넘긴다. 
- 다음으로, (b0 * a1) + c1 를 계산한 결과를 아래에 적고, 이러한 과정을 끝까지 반복한다.
- 그러면 결국, b0 * (a(n - 1) .. a0) 를 계산한 결과가, 아래 부분의 첫 번째 행에 나열된다.
- 마찬가지로, b1에 대해서도 (a(n - 1) .. a0) 를 곱한 결과를, 두 번째 행에 나열할 수 있다.
- 이 때, b1은 10의 자리 숫자이므로, 맨 오른쪽에 0을 1개 채운 후에, 결과를 나열해야 한다.
- 이를 반복하면, (n - 1) 개의 0과 b(n - 1) * (a(n - 1) .. a0) 의 값이 마지막 행에 나열된다.
- 그런 다음, 1의 자리에 있는 값들의 합을 아래에 적고, 올림수가 있으면 다음 자리로 넘긴다.
- 그리고, 올림수를 포함하여, 10의 자리에 있는 값들의 합을 구하고, 그 값을 아래에 적는다.
- 이러한 과정을 마지막 자리수까지 반복해서 얻은 결과는, 결국, A와 B를 곱한 결과가 된다.

## 1-3. 연산 횟수 파악

이번에는, n자리 숫자 2개를 곱하기 위해서, 몇 번의 기본 연산(곱셈, 덧셈) 이 필요한지 파악해보자.

```
  a(n - 1) a(n - 2) ... a1 a0
* b(n - 1) b(n - 2) ... b1 b0
─────────────────────────────
       b0 * (a(n - 1) ... a0)   -> n ┐
       b1 * (a(n - 1) ... a0)   -> n │
                                     ├ n * n = n^2
              ፧                   ፧   │
                                     │
 b(n - 1) * (a(n - 1) ... a0)   -> n ┘
─────────────────────────────
[]        ...        [] [] []  <-  A * B
↓                    ↓  ↓    ↘
n         ...        n  n   (n - 1) ≈ n
└──────────────────┬──────────────────┘
            2n * n = (2 * n^2)
```

- 첫 번째 행을 완성하려면, b0 * a0, b0 * a1, .. b0 * a(n - 1) 로, 총 n번의 곱셈이 필요하다.
- 다른 모든 행에 대해서도 n번의 곱셈을 수행해야 하므로, n * n = n^2 번의 곱셈이 필요하다.
- 그런 다음, 첫 번째 열에 있는 n개의 값을 다 더해야 하므로, 총 (n - 1) 번의 덧셈이 필요하다.
- 하지만, 올림수가 있는 열의 경우, (n + 1) 개의 값을 더해야 하므로, 총 n번의 덧셈이 필요하다.
- 이 때, n자리 숫자 2개의 곱셈에 대한 결과, 그것이 가질 수 있는 자리수의 최대 크기는 2n이다.
- 여기서, (n - 1) 를 n으로 취급하면, 필요한 덧셈의 횟수는, 총 2n * n = (2 * n^2) 번이 된다.

## 1-4. 정리

이 때, 덧셈과 곱셈은 적당한 크기의 두 수에 대한 기본 연산이므로, 상수 시간, 즉, 단위 시간이 걸린다.

- n^2 + (2 * n^2) = (3 * n^2) 번의 곱셈/덧셈을 수행했으므로, 이는, O(n^2) 이라고 할 수 있다.
- 즉, 'n자리 숫자 2개의 곱셈은 O(n^2) 만큼의 시간으로 충분히 계산할 수 있다.' 라고 할 수 있다.

<br>

이는, 학교에서 배웠던 곱셈 알고리즘을 사용했을 때에 필요한 수행 시간이 O(n^2) 임을 의미한다.

- 하지만, '큰 수 곱셈' 문제의 목적은, 이 방법보다 더 효율적이고, 더 빠른 방법을 찾는 것이다.
- 이 때, '그러한 방법이 있다.' 라는 것을 보이는 것이 바로, 카라추바의 분할 정복 알고리즘이다.

# 2. 카라추바 알고리즘(분할)

이전과 마찬가지로, n자리 숫자 2개를 곱하는 상황이라 가정하고, 카라추바 알고리즘에 대해 살펴보자.

> 입력으로는, n자리의 두 수, u와 v가 주어지며, 이들을 O(n^2) 보다 더 빠르게 곱하는 것이 목표다.

## 2-1. 작은 문제로 나누기

```
u = a(n - 1) a(n - 2) ... a(n / 2) a((n / 2) - 1) ... a1 a0 = x * 10^(n / 2) + y
    └─────────────┬──────────────┘ └───────────┬──────────┘
                  x                            y

v = b(n - 1) b(n - 2) ... b(n / 2) b((n / 2) - 1) ... b1 b0 = w * 10^(n / 2) + z
    └─────────────┬──────────────┘ └───────────┬──────────┘
                  w                            z

u * v = (x * 10^(n / 2) + y) * (w * 10^(n / 2) + z)
      = (xw * 10^n) + (xz + yw) * 10^(n / 2) + yz
```

- 우선, 입력으로 주어진 n자리 숫자 u를, 가운데를 기준으로 반씩 나눈 다음, 각각을 x, y라고 한다.
- 이 때, x라는 숫자는 (n / 2) 자리에서 시작하기 때문에, x의 실제 크기는 x * 10^(n / 2) 이 된다.
- 따라서, n자리 숫자인 u의 실제 크기는, u = x * 10^(n / 2) + y 라는 등식으로 표현할 수도 있다.
- 마찬가지로, v를 반으로 나눠, 각각을 w, z라고 하면, v = w * 10^(n / 2) + z 로 표현할 수 있다.
- 그러므로, u와 v를 곱한 것은, (x * 10^(n / 2) + y) * (w * 10^(n / 2) + z) 로도 표현할 수 있다.
- 또, 이렇게 구한 등식을 풀어보면, u * v = (xw * 10^n) + (xz + yw) * 10^(n / 2) + yz 가 된다.

## 2-2. 

여기서 중요한 것은, u * v 를 계산하기 위해서는, xw, xz, yw, yz를 모두 계산해야 한다는 것이다.

- 이 때, xw = x * w 이며, x의 자리수와 w의 자리수는 모두, n을 절반으로 나눈 (n / 2) 이 된다.
- 다시 말해, xw는 x와 w를 곱하는 것과 같으며, 이는 (n / 2) 자리 숫자 2개를 곱하는 것과 같다.
- 또한, xw와 마찬가지로, xz, yw, yz도, (n / 2) 자리 숫자 2개를 곱하는 것과 같다고 할 수 있다.

<br>

그런 다음에, x와 w를 곱한 다음에 어떻게 될까?

10^n 을 곱한다는 것은 뒤에다가 0을 n개 붙이는 것과 마찬가지다.

그 다음에, (xz + yw) * 10^(n / 2) 에서는 0을 (n / 2) 개 붙이는 것과 마찬가지이고, yz에서는 아무것도 하지 않는다.

그런 다음에, 다 더하면 되는 것이다.

결국은 이것은 어떻게 하냐면, n자리 숫자 두 개를 곱하는 것은 (n / 2) 자리 숫자 두 개를 곱하는 문제가 xw, xz, yw, yz, 즉, 4개를 계산한 다음에, 그 다음에 0을 n개 붙이고, 0을 (n / 2) 개 붙이고, 그렇게 해서 만든 각각의 항을 다 더하면 되는 것이다.

그러면, 그렇게 하기 위한 수행 시간을 계산할 수 있다.

## 2-3. 

T(n) 은 n자리 숫자 두 개를 곱하는 데 필요한 기본 곱셈과 덧셈 수라고 가정하면 된다.

T(n) 은 xw, xz, yw, yz를 계산해야 하는 것이다. 재귀적으로 말이다.

(n / 2) 자리 숫자 두 개를 곱하는 것이기 때문이다.

(n / 2) 자리 숫자 두 개를 곱하는 데 필요한 기본 연산의 횟수는 T(n / 2) 이고, 그것을 xw, xz, yw, yz에 대해 총 4번 반복한다.

그렇게 해서 계산이 되었으면, xw에는 0을 n개 붙이고, xz와 yw는 서로 더한다.

xz와 yw를 더하는 것은, x는 (n / 2) 자리이고, z는 (n / 2) 자리다.

이들을 곱하면, 곱한 결과는 n자리가 된다.

yw도 마찬가지로 n자리가 되는데, n자리와 n자리르 더하는 것은 n만큼의 시간이 걸린다.

n번을 덧셈하는 것과 마찬가지이고, 거기다가 (n / 2) 정도에 0을 붙이는 것이다.

결국은 xw * 10^n 과 (xz + yw) * 10^(n / 2) 과 yz라는 각각의 항을 준비하는 것은 추가적으로 n번의 0을 붙이고, n번의 덧셈과 (n / 2) 정도의 0을 붙이는 것이므로, 결국, 전부 합해서 3n 정도 이하의 추가 연산을 하면 되는 것이다.

그 다음에, 이렇게 만들어진 3개의 항을 최종적으로 다 더해야 하는 것이다.

더하는 데에는 몇 번의 연산이 필요할까?

xw * 10^n 은 총 몇 자리일까?

xw는 n자리이고, 거기에 0을 n개 붙였으니까, 2n자리가 된다.

따라서, 모든 항이 아무리 커도 2n자리를 넘지 않는다.

2n자리 3개를 더하는 데에는, 3n 정도의 덧셈으로 충분히 가능하다.

그래서 결국, 재귀적으로 (n / 2) 자리 숫자를 두 개를 곱하는 것을 4번 반복한 다음에, 추가로 연산하는 것은 상수  c에 n을 곱한 만큼의 연산만 하면 되는 것이다.

그러면, T(n) = 4T(n / 2) + cn 을 한 번 풀어보자.

이 점화식을 풀 때는, 우선, T(1) = c 라고 하고, 지금까지 한 것처럼, n이 계속 반씩 줄어들도록 점화식이 구성되어있기 때문에, n = 2^k 라고 가정을 하는 것이다.

그러면, T(n) = 4T(n / 2) + cn = (4^2 * T(n / 2^2)) + 2cn + cn 이 되고, 이것을 계속 반복하면, (4^k * T(n / 2^k)) + cn(1 + 2 + .. + 2^(k - 1)) 이 될 것이다.

이것을 다 계산해보면, O(4^k) 이 된다.

4^k 은 n^2 이다.

왜냐하면, n = 2^k, 즉, 2^k = n 이며, 각 항에 제곱을 하면, (2^k)^2 = n^2, 즉, 4^k = n^2 이 된다.

## 2-4.

그래서, 이렇게, n * n 의 문제를 절반 크기의 문제 4개로 쪼개서 풀었더니, O(n^2) 이 된 것이다.

그러면, 앞에서 학교 다닐 때 배웠던 기본적인 곱셈 연산의 O(n^2) 시간 알고리즘하고 다를게 없는 것이다.

분할 정복을 했는데, 똑같이 O(n^2) 만큼 하는 것이다.

나쁘진 않지만, 가장 간단한 방법도 O(n^2) 인데, 분할 정복으로 복잡하게 했음에도 그것보다 더 좋은, 빠른 시간이 걸리지 않았다는 것이다.

그래서, 카라추바는 생각을 했다.

아, 여기서 좀 더, 줄일 수 있는 방법은 없을까?

# 3. 카라추바 알고리즘(정복)

카라추바는 어떻게 생각을 했냐하면, 앞에서 생각했던 그 분할 정복 알고리즘은, 원래 문제를 절반으로 쪼갠 것을 4번 풀었다.

그러면, 원래 문제를 절반 크기의 문제 4개를 풀지 않고, 더 작은 개수를 풀면, 예를 들어, 3개로 풀 수 있으면, 더 빨리 할 수 있을 것이다.

그래서, 그 아이디어에 입각해서, 어떻게 생각했냐면, 이렇게 생각한 것이다.

여기서 우리가 반쪽짜리 문제가 xw, xz, yw, yz로 총 4개가 있다.

그래서, 원래 문제를 풀기 위해서는 절반 크기의 문제를 4개를 풀어야되는 것이다.

그런데, 그런 방식 대신에, xz와 yw를 어떻게 표현할 수 있냐하면, 이렇게 표현할 수 있다.

xz + yw = (x + y)(w + z) - xw - yz 가 나온다.

왜냐하면, (x + y)(w + z) - xw - yz = xw + xz + yw + yz - xw - yz = xz + yw 이기 때문이다.

그러면, 위에 있는 식은 u * v = (xw * 10^n) + (((x + y)(w + z) - xw - yz) * 10^(n / 2)) + yz 로 쓸 수 있다.

이전 식에 있던 xz + yw 를 ((x + y)(w + z) - xw - yz) 로 바꾼 것이며, 나머지는 똑같다.

그러면, 어떻게 생각할 수 있냐면, 여기서 (n / 2) 자리와 (n / 2) 자리를 곱하는 것이 xw, yz에서 2번은 등장한다.

그리고나서는, (x + y)(w + z) 를 계산해야 한다.

((x + y)(w + z) - xw - yz) 의 xw는 사실, (xw * 10^n) 의 xw와 똑같다.

따라서, 이것을 다시 구하지 않고, 그대로 다시 사용하는 것이고, yz도 마찬가지로, 다른 항에서 구했으므로, 그대로 사용하면 된다.

따라서, (x + y)(w + z) 를 계산하면 되는 것이다.

x + y 는 (n / 2) 자리와 (n / 2) 자리를 더하는 것이므로, 더한 값은 아무리 커도 (n / 2) + 1 자리가 된다.

w + z 도, (n / 2) + 1 자리가 된다.

그러면, n자리와 n자리를 곱하는 문제가, (n / 2) 자리와 (n / 2) 자리를 곱하는 문제가 2가지라고 말할 수 있다.

이 2가지는, xw와 yz를 구하기 위한, 두 번의 (n / 2) 자리와 (n / 2) 자리 곱하는 것을 해야 하는 것이다.

그리고, (x + y)(w + z) 를 구하기 위해, (n / 2) + 1 자리와 (n / 2) + 1 자리를 곱하는 것을 해야 한다.

그 다음에, 그렇게 해서 구한 (x + y)(w + z) 에서 xw, yz 를 빼고, xw에 0을 n개 붙이고, (x + y)(w + z) 에서 xw, yz 를 뺀 것에 0을 2n개 붙인 다음에, 각각 다 더하고, 이런 것을 모두 합쳐봤자, 추가적으로 하는 것은 n에 비례하는 기본 덧셈과 곱셈을 어떤 횟수만큼 하는 것이므로, 이것을 다시 표현하면, n자리와 n자리를 곱하는 데 걸리는 시간을 T(n) 이라고 하면, (n / 2) 자리와 (n / 2) 자리를 곱하는 데 걸리는 시간은 T(n / 2) 이고, 그것을 두 번 해야 하므로, 2를 곱하고, (x + y)(w + z) 는 (n / 2) + 1 자리를 두 번 곱하는 것이니까, T((n / 2) + 1) 이 되고, 기본 덧셈과 곱셈 연산을 cn만큼 하는 것이다.

T(n) = 2T(n / 2) + T((n / 2) + 1) + cn 에서, T((n / 2) + 1) 의 (n / 2) + 1 은, 사실 여기서 + 1 이 없다고 생각하더라도, 전개해서 빅오 표기법으로 표현하면, 있을 때와 없을 때의 O() 에 대한 인자가 똑같다.

그러니까, 간단하게 만들기 위해서 T((n / 2) + 1) 을 + 1 을 없애서, T(n / 2) 으로 만들 것이다.

그러면, T(n) = 3T(n / 2) + cn 이 된다.

즉, 절반 크기의 문제를 3번 푸는 것이다.

앞에서 했던 T(n) = 4T(n / 2) + cn 와의 차이점은, T(n / 2) 이 몇 번 곱해져 있는지, 즉, 3과 4의 차이다.

당연히, 절반 크기의 문제를 4번 푸는 것보다는 절반 크기의 문제를 3번 푸는 것이 훨씬 빠를 것이다.

T(n) = 3T(n / 2) + cn 을 풀면, T(n) = 4T(n / 2) + cn = O(n^2) 이었다.

이 때, n^2 = n^log2(4) 이다.

왜냐하면 절반 크기의 문제를 4개씩 풀었을 때의 점화식은, O(n^log2(4)) 이며, 3개씩 풀었을 때는 O(n^log2(3)) 이 된다.

이 때, n^log2(3) < n^log2(4) 이므로, O(n^log2(3)) < O(n^2) 이라고 할 수 있다.

T(n) = 3T(n / 2) + cn 이라는 점화식이 왜 O(n^log2(3)) 이 되는지는 온라인 수업 시간에 설명할 것이다.

```
T(1) = c, n = 2^k

T(n) = 3T(n / 2) + cn
     = (3^2 * T(n / 2^2)) + ((3 / 2) * cn) + cn
     = (3^2 * T(n / 2^2)) + (cn * (1 + (3 / 2)))
     = (3^2 * (3T(n / 2^3) + (c * (n / 2^2)))) + (cn * (1 + (3 / 2)))
     = (3^3 * T(n / 2^3)) + (3^2 * (c * (n / 2^2))) + (cn * (1 + (3 / 2)))
     = (3^3 * T(n / 2^3)) + ((3^2 / 2^2) * cn) + (cn * (1 + (3 / 2)))
     = (3^3 * T(n / 2^3)) + (cn * (1 + (3 / 2) + (3 / 2)^2))
       ...
     = (3^k * T(n / 2^k)) + (cn * (1 + (3 / 2) + (3 / 2)^2 + ... + (3 / 2)^(k - 1)))
     = O(3^k)
```

```

3^log2(n) = (2^log2(3))^log2(n) = 2^(log2(3) * log2(n))

log2(3^log2(n)) = log2(3) * log2(n) = log2(n) * log2(3) = log2(n^log2(3))

3^log2(n) = n^log2(3)

O(3^k) = O(3^log2(n)) = O(n^log2(3))
```
