# 1. 

```
분할 정복 알고리즘 수행 시간 T(n): 점화식(recurrence equation) 으로 표현!
```

앞에서 살펴본 분할 정복 알고리즘의 예들을 보면, 알고리즘들의 수행 시간들이 n에 관한 점화식으로 표현된다.

그러한 수행 시간을 T(n) 이라고 하면, T(n) 자체가 점화식으로 표현된다.

왜냐하면, n이라는 원래 사이즈의 문제를 여러 개로 쪼개서, 그것을 다시 같은 방법으로 풀기 때문에, 작은 문제에 대한 수행 시간이 더 작은 n에 대한 식으로 표현되어서, 결국은 점화식의 형태로 나타나는 것이다.

앞에서 설명하진 않았지만, 교재에도 있고, 강의 내용에도 있는 것이, 이진 탐색이라는 문제가 있다.

이진 탐색이라는 문제가 분할 정복 알고리즘의 대표적인 예 중에 하나이다.

이진 탐색은 Binary Search 라고 한다.

예를 들어, 친구와 내가 게임을 하는 상황에서, 친구가 1에서 100 사이의 어떤 수를 생각하고 있으면, 내가 질문을 해서 그 숫자를 맞히는데, 친구가 대답해줄 수 있는 것은 내가 짐작한 숫자보다 친구가 생각하고 있는 숫자가 더 크면 크다, 적으면 적다, 맞으면 맞다라고 얘기해주는 것이다.

그러면, 계속해서 질문을 하는데, 목표는 무엇이냐면, 최소한 적은 횟수의 질문만으로 친구가 생각하고 있는 숫자를 맞추는 것이다.

그럴 때, 여러분들의 입장에서는 어떤 전략으로 숫자를 짐작해 나가야 하느냐 하는 것이다.

그러면, 당연히 1부터 100까지 사이의 숫자 중에 하나를 친구가 생각하고 있기 때문에, 가장 안전한 방법은 1과 100 사이에 있는, 중간에 있는 숫자를 내가 먼저 짐작해보는 것이다.

만약에 50보다 친구가 짐작한 숫자가 더 적다면, 적다고 이야기할 것이고, 그러면 50부터 100 사이에는 친구가 생각하고 있는 숫자가 없는 것이다.

그러면, 50개의 후보를 버릴 수 있는 것이다.

그러면 1부터 49까지의 숫자들 중에 정답이 있기 때문에, 그 범위 가운데에 있는 숫자를 내가 말하면, 친구가 또 대답을 하고, 친구의 대답에 따라서 왼쪽 부분을 택할지, 오른쪽 부분을 택할지, 그렇게 진행하는 것이다.

이진 탐색은 똑같은 방식으로, 어떤 숫자들 또는 값들 사이에, 내가 원하는 값이 있는지 없는지를 탐색하는 것이다.

이 때의 목표는, 최소한의 비교 횟수만으로 탐색을 하는 것이다.

예를 들어, A에 2, 3, 9, 10, 17, 28, 31, 45라는 8개의 숫자가 있다고 가정해보자.

그러면 이 숫자는 현재 오름차순으로 정렬되어 있다.

정렬되어 있는 값들이 있는 리스트만을 대상으로 한다.

그러면 여기에 인덱스가 0, 1, 2, 3, 4, 5, 6, 7. 이렇게 인덱스 0부터 인덱스 7까지의 범위에 있는 값들 중에서 내가 원하는 또는 찾고 싶은 값 k가, 예를 들어, 31이었다고 해보자.

그러면, A에 있는 숫자들 중에서 31이 있는지를 알고싶은 것이다.

A에 있는 숫자들 중에서 몇 번째 숫자부터 이것을 찾아나가야할까?

우선, 가장 간단한 방법은 앞에서부터 비교해보는 것이다.

31과 A에 있는 모든 숫자를 각각 인덱스 순서대로 비교하는 것이다.

이러한 과정을 반복하다가, 31이 있는 6번째 인덱스에 도달하게 될 것이다.

그러면, 31과 같은 값이 A에 있다는 것이 확인된 것이다.

이러한 방법을 사용하면, n개의 숫자가 있을 때, 최악의 경우에는 n번의 비교가 필요하다.

그런데, 여기서의 힌트는, A에 있는 숫자들이 아무렇게나 순서에 상관없이 뒤죽박죽 들어있는 것이 아니고, 오름차순으로 정렬되어 있다는 것이 중요하다.

정렬되어 있다라는 정보를 이용하면, 이렇게 왼쪽부터 차례대로 비교해서 찾지 않고, 좀 더 효과적으로, 더 적은 비교만으로 찾을 수 있다.

앞에서 말했던, 친구와 내가 숫자 맞추기 게임을 하는 것과 똑같은 것이다.

1부터 100까지의 숫자들이 있는데, 당연히 정렬되어 있는 것이다.

그중에서 친구가 생각하고 있는 숫자를, 내가 짐작해서 맞추는 것이다.

그런데 A에 있는 숫자들이 정렬되어 있는 것이다.

그러면, 마찬가지로, 어느 위치에 있는 숫자부터 검색을 해야될지가 명확해지는 것이다.

그러면, 가장 앞에 있는 인덱스를 a라 하고, 가장 뒤에 있는 인덱스를 b라고 해보자.

처음 상태에서 a = 0, b = 7 이 된다.

즉, 0번째 인덱스부터 7번째 인덱스 사이에 있는 값 중에서 k가 있는지를 알고 싶은 것이다.

그러면, 처음에 비교를 해야될 것은 a 인덱스와 b 인덱스의 가운데에 있는 인덱스, 즉, 중간에 있는 값하고 비교하고 싶은 것이다.

그것은 a와 b를 합한 후에 정수 2로 나눗셈을 계산하면 된다.

(a + b) // 2 = (0 + 7) // 2 = 7 // 2 = 3

그러면 이것의 결과는 3이 된다.

그러면, 이렇게 얻은 3의 인덱스에 해당하는 값과 31을 비교하는 것이다.

이 때, 3번째 인덱스에 있는 값인 10보다 31이 더 크다.

31이 더 크기 때문에, 2부터 10까지, 즉, 0번째 인덱스부터 3번째 인덱스까지는 내가 원하는 값인 31이 없다는 것이다.

따라서 해당 범위는 확인하지 않아도 되는 것이다.

그러면, 현재 찾고자 하는 값은 위에서 구한 범위를 제외한 부분에 있을 것이다.

즉, 4번째 인덱스부터 7번째 인덱스의 사이에 31이 있는 것이다.

그러면, 이제 a = 4, b = 7 이 된다.

다시, 4번째 인덱스부터 7번째 인덱스 사이에 있는 것중에, 중간에 있는 값부터 비교하는 것이다.

그러면, 현재 범위의 중간 위치를 찾기 위해 계산을 해보면, (4 + 7) // 2 = 11 // 2 = 5 가 된다.

이번에는 5번째 인덱스에 있는 값인 28과 찾고자 하는 값인 31을 비교하면 된다.

28 < 31 이므로, 4번째 인덱스부터 5번째 인덱스까지는 내가 원하는 값이 없다는 뜻이다.

그러면 다시, 해당 범위를 제외한 6번째 인덱스부터 7번째 인덱스 사이에 찾고자 하는 값 31이 있을 수 있는 것이다.

그러면, a = 6, b = 7 이 될 것이다.

그런 다음, 해당 범위의 가운데 인덱스를 구하면 (6 + 7) // 2 = 13 // 2 = 6 이 된다.

즉, 6번째 인덱스에 있는 값인 31과 찾고자 하는 값인 31을 비교하면 된다.

이렇게, 31 = 31 이므로, 찾고자 하는 값의 위치를 찾은 것이다.

원하는 값의 인덱스를 얻었으므로, 이렇게 구한 인덱스 6을 반환하는 것이다.

이는 '6번째 인덱스에, 찾고자 하는 값인 k의 값이 들어있다.' 라는 것을 의미한다.

그리고, 주어진 범위 내에 찾고자 하는 값이 없을 수도 있다.

예를 들어, 내가 찾고 싶은 k의 값이 40이었다고 가정해보자.

k = 40 이었다고 하면, 이전 예시의 과정이 그대로 진행될 것이고, 마지막에 31과 40을 비교하게 될 것이다.

그러면, 31 < 40 이므로, 6번째 인덱스부터 7번째 인덱스까지는 내가 원하는 값이 없다는 뜻이다.

결국, 해당 범위를 제외한 7번째 인덱스부터 7번째 인덱스 사이에 찾고자 하는 값이 있을 것이다.

즉, a = 7, b = 7 이 될 것이다.

그런 다음, 해당 범위의 가운데 인덱스를 구하면 (7 + 7) // 2 = 14 // 2 = 7 이 된다.

다시, 7번째 인덱스에 있는 값 45와 찾고자 하는 값인 31을 비교하면 된다.

45 > 31 이므로, 7번째 인덱스부터 7번째 인덱스까지는 내가 원하는 값이 없다는 뜻이다.

현재 범위에서 중앙값보다 작은 범위는, 현재 범위의 시작 인덱스부터 (중앙값의 인덱스 - 1) 까지 이다.

즉, a = 7, b = (7 - 1) = 6 이 될 것이다.

이렇게, a > b 가 되었는데, 이는, 탐색 범위의 마지막 인덱스가 시작 인덱스보다 크다는 것을 의미한다.

이 때, 당연하게도, 6번째 인덱스부터 7번째 인덱스까지의 탐색 범위는 존재하지 않는다.

즉, A에는 찾고자 하는 숫자 k가 없다는 것을 의미한다.

그러면, -1을 반환한다.

이러한 방식으로 유지하는 것이다.

그래서 이것을 코드로 작성해보면 아래와 같다.

```
def bs(A, a, b, k):
    if a > b: return -1
    m = (a + b) // 2
    if A[m] == k: return m
    elif A[m] > k: return bs(A, a, m - 1, k)
    else: return bs(A, )
```

- 함수의 이름은 'binary search' 를 줄여서, 'bs' 라고 한다.
- 이 함수는 A에서, a번째 인덱스와 b번째 인덱스 사이에 있는 값 중에서 k라는 값의 위치를 찾는 함수다.
- 우선, 조건문을 이용해 a가 b보다 큰지 확인한다.
   - 만약 그렇다면, 찾고자 하는 숫자 k가 A에 없음을 의미하므로, -1을 반환하도록 한다.
- 그렇지 않다면, a와 b를 더한 값을 2로 나눗셈한 결과를 m이라는 변수에 할당한다.
   - 이 때, m은 현재 탐색 범위(a번째 인덱스부터 b번째 인덱스까지) 의 가운데 인덱스다.
- 다음으로, 조건문을 이용해 가운데 인덱스에 있는 값인 A[m] 과 k가 같은지 확인한다.
   - 만약 같다면, 원하는 값을 찾은 것이므로, 가운데 인덱스인 m을 반환하도록 한다.
- 그렇지 않다면, 조건문을 이용해 중앙값인 A[m] 이 찾고자 하는 값인 k보다 큰지를 확인한다.
   - 이는, 찾고자 하는 값 k가 현재의 중앙값보다 작은지를 확인하는 것이다.
   - 만약 그렇다면, 중앙값보다 작은 값 중에 k가 있다는 것이다.
   - 즉, 중앙값보다 작은 값, 즉, 원래의 절반 크기에 해당하는 탐색 범위에 대해 다시 이진 탐색을 반복해야 한다.
   - 이 때, 이진 탐색의 범위는 중앙값을 제외해야 한다.
   - 현재 탐색 범위의 시작 인덱스 a부터, 가운데 인덱스보다 1작은 (m - 1) 까지가 된다.
   - 따라서, 해당 범위에 대해 이진 탐색 알고리즘을 재귀적으로 호출하면 된다.
   - 그리고, 해당 재귀 호출에서 값을 찾는지 여부에 상관없이, 그대로 결과를 반환하면 된다.
- 그렇지 않다면, 찾고자 하는 값 k가 현재의 중앙값보다 크다는 것을 의미한다.
   - 즉, 위에서와 마찬가지로 중앙값보다 큰 범위에 대해 똑같이 재귀적으로 호출하면 된다.
   - 이 때, 이진 탐색 범위는 중앙값보다 1큰 값, 즉, (m + 1) 부터, 탐색 범위의 마지막 인덱스 b까지가 된다.

<br>

이렇게 하면, 재귀적으로 이진 탐색 알고리즘을 설계할 수 있는 것이다.

여기서 핵심은, 이진 탐색이 a부터 b까지의 범위에 대해서 찾는 것이 다음 재귀 호출에서는 중앙값보다 작은 범위에서 찾든지, 큰 범위에서 찾든지, 찾는 범위가 절반으로 줄어든다는 것이다.

아니면, 찾고자 하는 값을 바로 찾아서 즉시 반환하게 된다.

# 2.

이것을 점화식으로 표현해보자.

이진 탐색 알고리즘의 수행 시간을 T(n) 이라고 해보자.

```
T(n) = T(n / 2) + c
```

- 이 때, T(n) 은, 재귀 호출의 범위(중앙보다 큰 또는 작은)에 상관없이 둘 중에 하나만 호출하게 된다.
- 왜냐하면, `elif` 와 `else` 로 구분되기 때문이다.
- 즉, 어떤 것을 호출하든, 원래의 n개의 원소 중에서 찾는 문제가 (n / 2) 개 중에 값을 찾는 문제로 바뀌게 된다.
- 따라서, 절반 크기의 범위에 대해, 한 번만 재귀적으로 호출되는 것이다.
- 또, 재귀 호출 수행 전에 수행되는 덧셈, 나눗셈, 비교 연산 등의 기본 연산은 상수 횟수만큼만 수행된다.

<br>

이렇게 구성된 점화식 T(n) = T(n / 2) + c 을 풀면 된다.

앞에서 했었던 점화식들과 함게 살펴보자.

```
Best Case of Quick Select
T(n) = T(n / 2) + n

First Divide & Conquer Approach of Large Integer Multiplication
T(n) = (4 * T(n / 2)) + cn

Karatsuba Algorithm
T(n) = (3 * T(n / 2)) + cn
```

- Quick Select 알고리즘의 최선의 경우는 T(n) = T(n / 2) + n 이다.
- 큰 두 수를 곱할 때의 첫 번째 분할 정복 방법은 T(n) = (4 * T(n / 2)) + cn 이다.
- 두 번째 분할 정복 방법이었던 카라추바 알고리즘은 T(n) = (3 * T(n / 2)) + cn 이다.
- 이렇게, 굉장히 다양한 형태의 점화식들이 만들어진다.
- 그리고, 이러한 점화식들을 전개해서 풀 수 있다.
- 이러한 점화식들은 모두 n의 크기의 절반에 대해 재귀적으로 호출한다.
- 때문에, n이 몇 개의 (n / 2) 개의 문제, 즉, 반쪽 짜리 문제로 나누어지느냐가 다르다.
- 위에서 볼 수 있듯, 이러한 중간 크기 문제들의 개수는  1, 3, 4 등 다양한 개수가 될 수 있다.
- 물론, 이러한 재귀 호출 외에, 상수 횟수 또는 n의 크기에 비례하는 연산이 필요할 수 있다.
- 이외에도 다른 값들이 점화식에 추가될 수 있다.
- 하여튼, 이런 식으로 점화식이 만들어지는데, 이러한 점화식을 계속 전개해서 풀어도 된다.
- 하지만, 직관적으로, 즉, 그림으로 표현해서 쉽게 알 수 있는 방법도 있다.
- 그 내용은, 다음 수업에서 설명할 것이다.
