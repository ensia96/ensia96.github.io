## Microprocessor Cache

```
┌─────┐    address     ┌─────── Cache ───────┐        ┌────────────┐  address   ┌────────┐
|     | ─────────────→ |                     |  miss  |            | ─────────→ |        |
| CPU |                | - Cache Memory      | ─────→ | Memory Bus |            | Memory |
|     | ←─── data ───→ | - Cache Controller  |        |            | ←─ data ─→ |        |
└─────┘   (cache hit)  |                     |        └────────────┘            └────────┘
  ↑                    └─────────────────────┘              ↑
  |                               ↑                         |
  |                               | update                  |
  |                               |                         |
  └───────────────────────────────┴─────────────────────────┘
                             data
```

- CPU는 산술 논리 장치와 제어 장치, 레지스터로 구성된다. (요즘 CPU에는 캐시가 포함됨)
- CPU가 기억 장치에서 필요한 자료를 가져오려면, 버스라는 통신 체계의 도움이 필요하다.
- 물론, 버스로 연결하지 않아도 될 만큼 비교적 가까운 곳에 있는, 캐시를 활용할 수도 있다.
- 이 때, 캐시에서 자료를 가져오는 속도가 기억 장치에서 자료를 가져오는 속도보다 빠르다.
- 왜냐하면, 기억 장치의 경우 버스라는 매개를 거치는 데에 추가 시간이 소요되기 때문이다.  
  `(이외에도, 기억 장치와 캐시의 구조적 차이(SRAM, DRAM) 로 생기는 속도 차이도 존재한다.)`
- 때문에 CPU는, 필요한 자료를 찾을 때 기억 장치보다는 캐시를 먼저 확인하도록 설계된다.

<br>

- 필요한 자료가 캐시에 있는 상황은 캐시 히트(hit), 없는 상황은 캐시 미스(miss) 라고 한다.  
  `(hit = '캐시가 어떤 자료가 필요한지 맞혔다!', miss = '캐시가 필요한 자료를 놓쳤다.')`
- 이러한 히트/미스의 비율, 즉, 캐시가 효과적으로 사용되는 비율을 캐시 적중률이라고 한다.
- 캐시의 성능은 이러한 캐시 적중률과 속도, 기억 공간의 크기인 용량을 통해 표현되곤 한다.

## Virtual Memory(Memory Management)

```
          ┌─────┐
          | MPU |
          └─────┘
             |
             | virtual address
             ↓
          ┌─────┐
          | MMU |
          └─────┘
             |
             | physical address
             ↓
    ┌ Physical Memory ┐
    |─────────────────|
┌──→|        1        |
|   |─────────────────|
|   |        2        |←┐
|   |─────────────────| |
| ┌→|        3        | |
| | └─────────────────┘ |
| | ┌ Virtual Memory ─┐ |
└─┼─|        1        | |
  | |─────────────────| |
  └─|        3        | |
    |─────────────────| |
    |        2        |─┘
    └─────────────────┘
             ↑
         ┌──────┐
         | Disk |
         └──────┘
```

- 여기서 MMU는 'Memory Management Unit' 의 약자이며, 메모리 관리 장치라고 부른다.
- 메모리 관리 장치의 주된 역할은 가상 메모리 주소를 물리 메모리 주소로 변환하는 것이다.

## How to translate Virtual Adress to Physical Address!

```
┌─────────────┬─────────────┐
| Page Number | Page Offset | ← Virtual Address
└─────────────┴─────────────┘
       |             |
  Translation        |
       ↓             ↓
┌──────────────┬─────────────┐
| Frame Number | Page Offset | ← Physical Address
└──────────────┴─────────────┘
```
