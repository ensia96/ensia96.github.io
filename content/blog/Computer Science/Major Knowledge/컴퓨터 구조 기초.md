# 컴퓨터 구조

## 낮은 수준에서의 동작 원리

> 아래는, 참고 자료에는 없는 내용이지만, 정리할 필요성을 느끼고 있던 정보라서 정리했습니다.

<br>

- 컴퓨터에게 자료는 전기가 흐르는지 여부, 즉, 전기 신호이며, 이는 0과 1로 표현할 수 있다.
- 그리고 컴퓨터는, 여러 논리 회로를 통해 전기 신호를 조작하는 방식으로, 자료를 처리한다.
- 이렇게 논리 회로로 전기 신호를 조작하는 절차를 '컴퓨터가 연산을 수행한다.' 고 표현한다.
- 즉, 컴퓨터를 구성하는 회로에 전기 신호를 흘려보낼 때마다 연산이 한 번 수행되는 것이다.
- 이러한 이유로, 컴퓨터의 연산 수행 속도는 전기가 흐르는 주기에 영향을 받을 수밖에 없다.
- 여기서, 전기가 흐르는 주기는 클럭 속도라고 부르며, 보통, 주파수 단위인 'Hz' 로 표현한다.  
  `(참고로, 클럭 속도에 맞춰 회로에 전기 신호를 흘려보내는 장치의 이름은 '클럭 생성기' 이다.)`

<br>

- 컴퓨터 기억 장치는 '래치' 라는 특수한 형태의 회로를 이용하여 자료를 저장하는 장치이다.
- 래치는 0과 1을 기억하는 회로이며, 이러한 래치를 나열하면, 더 큰 자료를 기억할 수 있다.
- 2진수 형태의 어떤 자료를 a라고 했을 때, 2진수 a의 각 자리수가, 래치에 저장되는 것이다.
- 하나의 자료를 저장하는 이러한 래치 집합들을 모으고, 그것을 나열하면, 기억 장치가 된다.
- 간단히 요약하면, 컴퓨터 기억 장치는 '자료를 저장하는 래치 집합' 의 집합이라 할 수 있다.

## Microprocessor Cache

```
┌─────┐    address     ┌─────── Cache ───────┐        ┌────────────┐  address   ┌────────┐
|     | ─────────────→ |                     |  miss  |            | ─────────→ |        |
| MPU |                | - Cache Memory      | ─────→ | Memory Bus |            | Memory |
|     | ←─── data ───→ | - Cache Controller  |        |            | ←─ data ─→ |        |
└─────┘   (cache hit)  |                     |        └────────────┘            └────────┘
  ↑                    └─────────────────────┘              ↑
  |                               ↑                         |
  |                               | update                  |
  |                               |                         |
  └───────────────────────────────┴─────────────────────────┘
                             data
```

- 여기서 MPU는 'Micro Processing Unit' 의 약자로, 마이크로프로세서라고 부르기도 한다.
- 이러한 MPU는 하나의 집적 회로로 구성되어 있으며 주어진 연산을 처리하는 역할을 한다.
