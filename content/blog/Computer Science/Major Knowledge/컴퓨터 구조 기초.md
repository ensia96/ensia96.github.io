## Microprocessor Cache

```
┌─────┐    address     ┌─────── Cache ───────┐        ┌────────────┐  address   ┌────────┐
|     | ─────────────→ |                     |  miss  |            | ─────────→ |        |
| CPU |                | - Cache Memory      | ─────→ | Memory Bus |            | Memory |
|     | ←─── data ───→ | - Cache Controller  |        |            | ←─ data ─→ |        |
└─────┘   (cache hit)  |                     |        └────────────┘            └────────┘
  ↑                    └─────────────────────┘              ↑
  |                               ↑                         |
  |                               | update                  |
  |                               |                         |
  └───────────────────────────────┴─────────────────────────┘
                             data
```

- 필요한 자료가 캐시에 있는 상황은 캐시 히트(hit), 없는 상황은 캐시 미스(miss) 라고 한다.
- 캐시 미스가 발생하면, CPU는 버스의 도움을 받아, 기억 장치에서 필요한 자료를 가져온다.

## Virtual Memory(Memory Management)

```
┌─────┐  Virtual Address  ┌─────┐  Physical Address  ┌────────┐
| MPU | ────────────────→ | MMU | ─────────────────→ | Memory |
└─────┘                   └─────┘                    └────────┘
```

- 여기서 MMU는 'Memory Management Unit' 의 약자이며, 메모리 관리 장치라고 부른다.
- 이러한 메모리 관리 장치는, 가상 주소(논리 주소) 를 물리 주소로 변환해주는 역할을 한다.

## How to translate Virtual Adress to Physical Address!

```
 ┌─────────────┬────────┐
 | Page Number | Offset | ← Virtual Address
 └─────────────┴────────┘
        |          |
   Translation     |
        ↓          ↓
┌──────────────┬────────┐
| Frame Number | Offset | ← Physical Address
└──────────────┴────────┘
```

- 기억 장치의 단위 공간인 프레임을, 페이지로 취급 및 관리하는 기법을 페이징이라고 한다.
- 가상 주소와 물리 주소는 각각 페이지 번호와 오프셋, 프레임 번호와 오프셋으로 구성된다.
- 프레임 번호는, 자료가, 기억 장치의 어느 프레임에 저장되어 있는지를 나타내는 주소이다.
- 오프셋은, 단위 공간을 기준으로, 자료가 어느 곳에 저장되어 있는지를 나타내는 주소이다.

### 1. Basic Translation

```
   Virtual Address
(Page Number, Offset)
   ┌───┬────────┐
   | 3 | Offset | ────────────────────────────────────┐
   └───┴────────┘                                     |
     |                                                |
     |               Page Table                       |
     |     ┌─────────────┬──────────────┐             |
     |     | Page Number | Frame Number |             |
     |     |─────────────|──────────────|             |
     |  ┌→ |     ...     |     ...      |             ↓
     |  |  |─────────────|──────────────|    ┌────┬────────┐
     └──┼→ |      3      |      17      | ─→ | 17 | Offset | ←    Physical Address   
        |  |─────────────|──────────────|    └────┴────────┘   (Frame Number, Offset)
        └→ |     ...     |     ...      |
           └─────────────┴──────────────┘
```

- 가상 주소를 물리 주소로 바꿔주려면, 우선 페이지 번호를 프레임 번호로 바꿔주어야 한다.
- 이 때, 페이지 번호에 대응되는 프레임 번호는 페이지 테이블이라는 자료 구조에 저장된다.
- 이러한 페이지 테이블을 구성하는 각 칸 또는 자료 쌍은 페이지 테이블 엔트리라고 부른다.
