# 0. 시작하기에 앞서,

이번 수업에서는 프로세서에 대한 내용을 다룰 것이기 때문에,  
아마도 전체 강의 중에서 가장 복잡한 수업이 될 것이다.

지금까지의 수업에서 구성했던 요소들을 정리해보면 아래와 같다.

- 2진수를 받아 계산을 수행하는 **산술 논리 장치(ALU)**
- 단일 값을 저장하는 데에 유용한 선형적인 메모리 덩어리인 **레지스터(Register)**
- 많은 숫자를 각각 다른 주소에 저장할 수 있는 **임의 접근 기억 장치(RAM)**

<br>

이것들을 종합해, 컴퓨터의 핵심 장치인 **'CPU'** 를 구성해볼 것이다.

> CPU(Central Processing Unit) : 중앙 처리 장치

# 1. 명령어

'MS Office', 'Safari' 와 같은 프로그램들은 일련의 작업들로 구성되어 있는데,  
CPU는 프로그램을 실행하기 위해 컴퓨터가 해야 할 일을 각 장치에 지시하는 역할을 한다.

이 때, CPU가 지시(instruct) 하는 내용을 **'명령어(instruction)'** 라고 한다.

- 덧셈, 뺄셈 등의 명령어가 입력되면, CPU는 ALU가 수학적 연산을 처리하도록 구성한다.
- 메모리 명령어가 입력되면, CPU는 기억 장치를 이용해 값을 읽거나 쓰는 동작을 수행한다.

# 2. 마이크로아키텍처

CPU에는 많은 부품이 있어 세부 구성을 파악하기엔 많이 복잡하기 때문에,  
모든 배선을 확인하기 보다는 기능 단위 블록을 하나씩 나열하면서 살펴볼 것이다.

이렇게, 추상적으로 표현된 CPU 구조를 **마이크로아키텍처(microarchitecture)** 라고 한다.

# 4. 제어 장치

각각의 decode 회로를 살펴보는 것은 너무 상세하다.

이미 하나의 예제를 살펴봤기 때문에, 다른 것들은 Control Unit 으로 묶어서 단순화할 것이다.

새로운 추상화 단계이다.

Control Unit 은 CPU의 다른 모든 파트를 지휘하기 때문에 오케스트라의 지휘자와 같다.

인출/해독/실행의 전체 사이클이 한 번 끝났으니, 이 모든 것을 다시 시작할 준비가 되었다.

fetch phase 부터 시작한다.

명령어 주소 레지스터는 1을 갖고 있고, RAM은 주소 1번지에 저장된 00011111 을 보낸다.

decode phase 에서, 0001은 'LOAD_B' 명령이고, 이것은 RAM에서 값을 Register B로 이동시키는 것이다.

이번에 메모리 위치는 1111(10진수로 15) 이다.

이제 execute phase 다.

Control Unit 은 RAM의 15번지를 read할 수 있도록 설정하고,  
Register B는 정보를 받을 수 있도록 설정한다.

빙고, 이제 막 00001110(10진수 14) 를 Register B에 저장했다.

마지막으로 할 일은 명령어 주소 레지스터를 1 증가시키고, 또 하나의 사이클을 완료한다.

# 5. ALU가 필요한 명령

다음 명령은 조금 다르다.

우선 fetch 해보자.

10000100.

opcode 1000 은 ADD 명령이다.

이 명령은 4bit RAM 주소 대신 2bit 짜리 2개 세트를 사용한다.

2bit는 4개의 값을 표현할 수 있으니까, 우리는 4개의 레지스터 중에 하나를 선택할 수 있다.

첫번째 세트는 01이고, Register B를 나타낸다.

다음은 00으로 Register A를 나타낸다.

따라서, '1000 01 00' 은 Register B의 값을 Register A의 값에 더하라는 명령이다.

이 명령을 수행하기 위해서, '5' 에서 배운 ALU를 합칠 것이다.

Control Unit은 올바른 레지스터를 선택해서 입력해주고,  
올바른 연산이 수행되도록 ALU를 설정해야 한다.

이 ADD 명령을 위해서 Control Unit은 Register B를 enable하고,  
저장된 값을 ALU의 첫번째 입력으로 보낸다.

또한 Register A를 활성화하여 ALU의 두번째 입력으로 보낸다.

앞서 말한 것처럼 ALU는 몇 개의 다른 연산을 수행할 수 있다.

따라서, Control Unit은 ADD opcode 를 전달해서 ADD 연산이 수행되도록 설정해야만 한다.

마지막으로, 연산 결과가 Register A에 저장되어야 한다.

하지만 바로 저장할 수는 없다.

이 value가 ALU로 되돌아와서 다시 더해질 수 있기 때문이다.

그래서, Control Unit을 내부 레지스터에 연산 결과를 임시로 저장하고,  
ALU를 끈 다음, 목적지 레지스터에 값을 저장한다.

이 경우에, 입력은 3과 14이므로 합은 17, 이진수로 00010001 이다.

이 값은 현재 Register A에 있다.

지난 번과 마찬가지로 마지막 할 일은 명령어 주소 레지스터를 1 증가시키는 것이다.

또 하나의 사이클이 완성되었다.

자, 이제 마지막 명령 01001101 을 fetch 해보자.

해독해보면, 0100은 STORE_A 명령이고, RAM 주소는 13이다.

마찬가지로 RAM 모듈에 주소를 전달하지만, read enable 대신에 write enable 을 한다.

동시에 Register A도 read enable하여 data 선에 Register A에 저장된 값이 통과하게 한다.

# 6. 클럭 속도

축하한다.

지금 막 우리는 첫 컴퓨터 프로그램을 실행했다.

메모리로부터 2개의 값을 옮겨와서, 서로 합친 후에 결과를 메모리로 다시 저장하는 프로그램이었다.

물론 선생님이 fetch, decode, execute phase 의 각 단계를 설명하면서 CPU를 수동으로 전환하고 있었다.

하지만, 모든 컴퓨터 속에 mini Carrie Anne 이 들어가 있을 수는 없다.

그래서 CPU를 착착 작동시킬 수 있게 clock 이라고 하는 부품을 추가할 것이다.

이름에서 알 수 있듯이, clock은 정확하고 규칙적인 시간에 전기 신호를 발생한다.

이 신호는 Control Unit 이 CPU의 내부 동작을 완벽하게 발 맞춰 진행시킬 수 있게 해준다.

마치 로마 갤리선의 앞에서 리드미컬하게 북을 치는 사람처럼 말이다.

그럼, 모든 노 젓는 사람이 동시에 박자를 맞출 수가 있다.

물론 무작정 빨리 갈 수는 없다.

아무리 전자라도 전선을 통과해서 신호가 안정될 때까지 시간이 필요하니까 말이다.

CPU가 fetch-decode-execute 사이클의 각 단계를 수행하는 속도를 clock speed 라고 한다.

이 속도는 주파수 단위인 헤르츠(Hz) 로 표시한다.

1 헤르츠는 1초에 1사이클이라는 의미이다.

선생님이 LOAD, LOAD, ADD, STORE 4개의 명령에 대해서 설명하는데 대략 6분 정도가 걸렸다.

이 말은, 선생님의 실제적인 clock speed 는 대략 0.03Hz 라는 것이다.

선생님이 뛰어난 컴퓨터는 아니란 것을 인정하지만,  
수학에 재능있는 사람은 머리 속으로 1초에 계산 하나 씩 할수 있을 것이다.

1Hz 의 속도로 말이다.

# 7. 최초의 CPU

최초의 단일칩 CPU는 1971년에 출시된 Intel 4004라는 4bit CPU 이다.

이 CPU의 마이크로아키텍쳐는 우리가 예제로 사용하고 있는 것과 상당히 비슷하다.

최초의 프로세서임에도 불구하고 놀랍게도 740kHz로 작동했다.

1초에 74만 사이클이 수행된다는 것이다.

엄청 빠르다고 생각할 수 있지만, 요즘 우리가 사용하는 프로세서와 비교하면 아무것도 아니다.

1메가헤르츠(MHz) 는 1초에 백만 clock 사이클이고,  
이 비디오를 보고 있는 컴퓨터나 스마트폰은 몇 기가헤르츠(GHz) 로 작동한다.

매 1초마다 수십억개의 CPU 사이클이 작동하는 것이다.

# 8. 오버클럭과 언더클럭

혹시 컴퓨터를 overclocking 해서 사용한다고 들은 적이 있을 지도 모른다.

clock 속도를 올리면, CPU 의 속도를 올릴 수 있다.

마치 로마 갤리선의 북치는 사람이 다른 배와 충돌하기 위해 속도를 올리는 것 처럼 말이다.

칩 제조사는 약간의 overclocking 이 가능하도록 CPU 를 설계한다.

하지만, 너무 많이 하면 CPU 가 과열되거나 신호가 안정되는 속도보다 clock 이 빨라질 수 있다.

underclocking 에 대해서 많이 들어보진 못했을 것 같지만, 이것도 유용하다.

때때로 프로세서가 최고 속도로 작동할 필요가 없을 때가 있다.

사용자가 자리를 떠났거나, 별다른 프로그램을 실행하지 않는 경우  
CPU 속도를 느리게 해서, 전력 소비를 많이 줄일 수가 있다.

이것은 배터리로 동작하는 노트북이나 스마트폰에서는 아주 중요한 일이다.

# 9. 동적 주파수 스케일링

이런 요구사항을 만족하기 위해, 많은 요즘 프로세서들은 상황에 맞게 clock speed 를 높이거나 낮출 수 있다.

이것을 'dynamic frequency scaling(동적 주파수 스케일링)' 이라고 한다.

자 이제 clock 을 추가해서 우리의 CPU 가 완성되었다.

이제 이것들을 상자에 넣고, 하나의 부품으로 만들어보자.
