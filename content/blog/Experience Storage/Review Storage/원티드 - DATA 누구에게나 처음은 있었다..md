# 1. 데이터 분석가가 바라보는 매력적인 이력서의 공통점

> 출처 : https://www.wanted.co.kr/events/careertalk08

> 발표자 : 원티드랩 데이터분석가 이상인

> 주제 : 데이터 관련 업무에서 중요하다고 생각하는 점들을 잘 어필하는 매력적인 이력서와 포트폴리오

## 1-1. 데이터 분석가에 대한 정의

> 발표자의 개인 경험에 기반한 내용임

- 데이터 분석가, 사이언티스트, 엔지니어 등 데이터 관련 직군은 그 경계가 모호함
- 실제 현업에서 일하는 사람들도, 본인만의 기준으로 각 직군의 역할을 나누기도 함

<details><summary>데이터 사이언티스트와 데이터 분석가</summary>

| 데이터 사이언티스트 | 데이터 분석가 |
|---|---|
| - 데이터 엔지니어 의존도가 비교적 낮음<br>- 모델링 생성 및 고도화<br>- 호흡이 긴 업무 | - 서비스/제품 분석 전반 참여<br>- 비교적 업무 호흡이 짧음 |

</details>

## 1-2. 데이터 분석가가 하는 일

```
- 타 부서에서의 데이터 추출/분석
- 가설에 기반한 데이터 분석
- 서비스 개선을 위한 제품/사업 분석
- 대시보드 생성 및 관리
- 분석 툴, SQL 활용 가이드(및 교육) 등
```

- 업무 내용 자체는 그렇게 어렵지 않은 편이기 때문에, 누구나 할 수 있을 것임
- 프로그래밍 언어는 R, Python, 대시보드는 tableau, Power BI 등을 사용함
- 하지만, 이러한 기술적인 부분 외에도 데이터 분석가가 갖춰야 할 요소가 있음

<details><summary>데이터 추출 및 분석</summary>

> 데이터를 추출하여, 타 부서에 전달하는 업무

- 사내에서 쓰이는 분석 툴(GA, amplitude 등) 로 확인할 수 없는 부분이 요청됨
- 최신화나 꾸준한 관찰이 필요한 경우, 쿼리 생성, 자동화 등의 작업을 진행함  
  `(SQL, airflow, docker 등의 기술이 활용됨)`
- 일부 기업의 경우, 이러한 데이터 추출 관련 업무만 수행하는 직군도 있다고 함

```
- 요청자의 의도를 만족할 수 있는지
- 이외에 의사결정에 도움이 될만한 추가 데이터, 과거 인사이트는 무엇이 있는지
- 추가 분석이 필요한지
- 의사결정자가 제공된 데이터를 바탕으로 논리적인 인사이트를 도출하는지
```

</details>

<details><summary>서비스 개선을 위한 제품/사업 분석</summary>

- 제품 또는 사업의 변화 이후, 해당 변화가 서비스에 어떤 영향을 주었는지를 분석
- 작업 진행 순서는 보통 기획, 개발, 배포로 나뉘며, 단계마다 분석가의 역할이 다름

```
기획(개발 전)

- 프로젝트 목적에 맞게 지표를 설정했는가?
- 성과를 보기에 더 적합한 지표, 추가해야 할 지표는 없는가?
- 상위 KPI와 잘 부합하는가?
- 해당 프로젝트의 가치는 얼마나 되는가?

개발

- 성과 측정을 위한 데이터를 기록하는가?

배포(개발 후)

- 배포 후 얼마나 개선되었는가?
- 유의미한 개선이었는가?
- 배포 회에 다른 변수가 없는가?
- 퍼널에 병목이 있는가?
- 유저 그룹별로 지표 변화가 있는가?
```

</details>
