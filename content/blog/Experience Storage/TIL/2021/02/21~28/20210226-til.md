- 프로그래머스 알고리즘 문제를 풀었다.
   - '완주하지 못한 선수' 라는 해시 알고리즘 문제를 풀었다.
   - 배열 비교에 해시를 적용하는 방법을 생각하느라 처음엔 조금 헤매였다.
- 주소값(key) 과 정보(value) 구성의 해시 테이블에 대해 공부했다.
   - Python 의 딕셔너리 자료형도 해시 테이블이란 것을 알게됐다.
      - 딕셔너리의 key 에 해시화 불가능한 값이 들어가지 못하는 이유가 있었다.
      - 주소값으로 사용되는 key 가 실제로는 해시값으로 처리되기 때문이였다.
- Python 의 collections 모듈의 Counter 객체와 hash() 함수에 대해 알게됐다.
   - collections 모듈은 전에 알고리즘 문제 풀이 중에 본 적이 있었는데, Counter 객체는 처음 봤다.
      - 컨테이너 자료형의 연산이 내부 값에 대해 처리되는 것이 신기했다.
      - 특정 배열(list) 를 Counter 클래스의 생성 구문에 인자로 넣으면,  
        값: 개수 의 형태를 포함하는 Counter 객체가 생성된다.
   - Python 의 내장 함수 hash 에 대해 알게됐다.
      - 객체 정보를 인자로 받아, 해당 정보의 해시값을 반환한다.
      - 이 때, 반환되는 해시값은 정수형 자료이다.
      - 해시화 불가능한 입력값이 인자로 오면 오류를 발생한다.
         - 딕셔너리, 리스트처럼 해시화 불가능한 값은 자료형은 안된다!
- JavaScript 의 기초 문법에 대해 공부했다.
   - 감소 연산자(--) 를 할당 구문에 사용하면 감소된 값이 할당되고,  
     해당 실행 과정에선 연산 전 값으로 취급된다는 것을 알게됐다.
   - find 메서드에 두 번째 인자를 넘기면,  
     그 인자의 값을 실행 과정의 'this' 로 취급한다는 것을 알게됐다.
      - 실행 기준값이 되기 때문에 해당 실행 단위에서 가장 먼저 접근한다.
