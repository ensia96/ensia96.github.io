# 오늘 한 일

- Crash Course 내용 정리
   - '4. 행렬' 의 시작 부분까지 정리했다.
   - 회로 구성하고, 이미지 만드는 데에 시간을 너무 많이 썼다..
- 래치에 대해 공부했다.
   - 단일 비트의 정보를 기억하는 회로다.
   - 특정한 값에 달라붙어서(latches onto) 값을 저장한다고 해서 'Latch' 라고 부른다.
   - set, reset 2개의 입력을 받고, 1개의 값을 출력한다.
   - 동작 방식은
     ['1-3. 회로 구성'](/Computer Science/Crash Course/6. 레지스터와 RAM/#1-3-회로-구성)
     에 정리해뒀다.
- 게이트 래치에 대해 공부했다.
   - 래치의 입력 부분에 회로를 추가해서 구상한다.
   - 정보 입력(data input) 과 쓰기 가능(read enable) 을 입력받고,  
     정보 출력(data output) 을 출력한다.
   - 논리 회로(logic gate) 의 입력을 열고 닫을 수 있어서 'Gated Latch' 라고 부른다.
   - 기본 래치에 비해, 입력을 조합하고 관리하기 편하다.
   - 동작 방식은
     ['2-2. 동작 방식'](/Computer Science/Crash Course/6. 레지스터와 RAM/#2-2-동작-방식)
     에 정리해뒀다.
- 레지스터에 대해 공부했다.
   - 여러 개의 래치로 구성된 단위 정보 기억 장치다.
   - 사용된 래치의 개수를 레지스터의 너비라고 한다.
   - 여러 개의 래치에 각각 쓰기 가능선을 연결하는 것이 아니라,  
     하나의 회로를 추가해 한 번에 쓰기 기능을 제어한다.
   - 쓰기 가능 입력을 제어하는 선 하나와 래치 당 입/출력선이 1개씩 필요하다.
   - 래치를 나란히 배치하면 구성에 필요한 전선의 수가 크게 늘어난다.
