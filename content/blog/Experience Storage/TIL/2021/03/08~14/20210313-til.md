# 오늘 한 일

- Crash Course 내용 정리
   - '7. 클럭' 까지 정리했다.
- 마이크로아키텍처에 대해 공부했다.
   - CPU의 전선 구성 등, 지나치게 세부적인 구성은 생략한다.
   - 각각의 핵심적인 단위 요소들과 간단한 연결 구성을 표기한다.
   - CPU의 운영에 필요한 내용들과 물리적인 설계가 정리되어 있다.  
     `(마이크로프로세서(CPU)의 기본 디자인이다.)`
- 간단한 CPU를 구성해봤다.
   - 값을 임시로 저장, 조작하기 위해 임의의 레지스터를 사용한다.
      - 실행할 명령어의 내용을 저장하기 위해
      - 실행 중인 명령어의 메모리 주소를 저장하기 위해
   - 명령어나 값을 읽고 쓰기 위해 RAM 모듈을 사용한다.
   - 다양한 연산을 처리하기 위해 ALU를 사용한다.
   - 각각의 장치들을 조작하기 위해 제어 장치를 사용한다.
- CPU가 동작하는 단위인 명령 주기에 대해 공부했다.
   - 메모리에서 명령을 읽어오는 과정은 '인출 단계' 다
   - 해당 명령어가 제어 장치에서 처리되는 과정은 '해석 단계' 다.
   - 제어 장치에 의해 CPU의 각 장치들이 실제로 작동하는 과정은 '실행 단계' 다.
   - 명령의 실행이 끝나면, 다음 명령을 수행하기 위해 명령어 주소를 갱신한다.
- CPU의 제어 장치에 대해 공부했다.
   - 입력받은 명령에 대해 CPU의 각 장치들이 동작할 수 있도록 지시하는 역할을 한다.
      - 여러 논리 회로들을 거쳐 여러 장치가 연결된 특정 전선에 신호를 출력한다.
      - 이것을 '제어 장치가 명령어를 번역(혹은 해독, 해석)한다' 라고 표현한다.
      - 이렇게 출력된 신호에 대한 각 장치들의 반응이 조합되어 작업이 수행된다.
   - 결과가 있는 연산이 수행되면, 그 결과를 임시로 저장할 수 있다.
   - 메모리에서 정보를 읽고 쓰는 등의 다양한 동작을 제어할 수 있다.
- CPU의 작업을 진행시키는 클럭에 대해 공부했다.
   - 규칙적인 간격으로 전기 신호를 발생시키는 장치다.
   - 전기 신호의 발생 여부에 의해 CPU의 작업이 진행된다.
      - 전기가 쭉 흐르기만 하면, 변화가 발생할 수 없다.
      - 전기가 멈췄다가 다시 흘러야 회로의 상태가 구분된다.
