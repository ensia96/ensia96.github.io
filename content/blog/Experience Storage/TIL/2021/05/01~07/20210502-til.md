# 오늘 한 일

- Crash Course 내용 정리
   - '20' 의 '배운 점, 느낀 점' 전까지 마무리했다.
- 단층 파일 시스템에 대해 공부했다.
   - 파일을 변경하는 작업을 위해선, 디렉토리 파일의 정보를 수정해야 한다.
   - 이렇게 디렉토리 파일을 유지 관리하는 것은 단순한 형태의 파일 시스템이다.
   - 이 때, 모든 파일이 하나의 계층에 저장된 평평한 구조의 파일 시스템을 단층 파일 시스템이라 한다.
   - 단층 파일 시스템은 파일을 연속적으로 저장하는 것이 아니라, 블록 단위로 저장한다.
      - 덕분에, 파일의 변경 작업을 처리할 수 있는 느슨한 공간이 생긴다.
   - 그리고, 각각의 파일을 큰 덩어리로 나눠서 여러 블록에 저장한다.
      - 블록을 할당하고 할당 해제하는 것으로 파일의 크기를 쉽게 조절할 수 있다.
- 파일 복구에 대해 공부했다.
   - 파일 시스템에서 특정 파일을 삭제할 때, 실제로는 정보가 지워지지 않는다.
   - 대신, 디렉토리 파일에서 해당 항목을 제거하여, 기존 정보를 유지한다.
   - 이러한 파일 시스템의 특징을 이용해 삭제된 파일을 복구할 수 있다.
      - 정확히는, '삭제되었다' 라기보다는 '삭제 표시되었다' 라고 할 수 있다.
      - 컴퓨터 법의학 팀에서도 파일 복구를 위해 이러한 특징을 이용하기도 한다.
- 단편화와 조각 모음에 대해 공부했다.
   - 파일이 생성, 삭제, 수정될 때, 블록이 뒤죽박죽으로 할당되기도 한다.
   - 이렇게 정보가 나뉘어서 저장되는 현상을 메모리 단편화라고 한다.
      - 자기 테이프와 같은 순차 접근 기억 장치에서는 심각한 문제가 된다.
      - 파일 하나를 메모리에 읽어들이기 위해, 많은 움직임이 필요해지기 때문이다.
   - 이러한 메모리 단편화를 해결하는 방법이 조각 모음이다.
   - 컴퓨터가 정보를 복사해, 파일의 블록들을 저장 장치에 다시 배치하는 절차로 진행된다.
   - 조각 모음이 끝나면, 파일의 정보가 담긴 블록들이 하나로 연결된다.
- 계층적 파일 시스템에 대해 공부했다.
   - 특정 폴더에 다른 파일과 폴더를 모아서 저장하는 방식을 계층적 파일 시스템이라 한다.
   - 이 때, 모든 디렉토리들의 최상위에 위치하는 디렉토리를 루트 디렉토리라고 한다.
   - 하위 파일과 폴더의 위치는 루트 디렉토리로부터 다양한 경로를 통해 확인할 수 있다.
   - 디렉토리 파일의 내용을 바꾸는 것만으로도 다른 파일이나 폴더를 쉽게 옮길 수 있다.
