# 오늘 한 일

- 자료 구조, 알고리즘 수업 정리
   - '알고리즘 선택문제 소개' 내용 정리를 마무리했다.
- 선택 문제와 최대값 찾기 문제, 상한과 하한에 대해 공부했다.
   - 입력의 크기 n에 대해, n개의 값과 1과 n 사이에 있는 k라는 값을 입력으로 받는다.
   - 최소한의 비교 횟수를 이용해서 n개의 값 중에 k번째로 작은 값을 찾아서 출력해야 한다.
   - k = n 인 경우, 첫 번째 값을 제외한 모든 항목을 하나씩 비교해 문제를 풀 수 있다.
      - n개의 값 중, n번째로 작은 값을 찾아야 하며, 이러한 문제를 최대값 찾기 문제라고 한다.
   - 이 때, 입력의 크기는 n이므로, (n - 1) 번의 비교로 문제를 풀 수 있다.
   - 이렇게, 어떠한 상황에서도 문제를 풀 수 있는 연산 횟수를 상한이라고 한다.
      - 최대값 찾기 문제에서 상한은 항상 최대값을 찾을 수 있는 비교 횟수인 (n - 1) 이다.
   - 반대로, 어떠한 문제를 푸는 데 필요한 최소한의 연산 횟수는 하한이라고 한다.
      - 최대값 찾기 문제에서 하한은 최대값을 찾기 위한 최소한의 비교 횟수인 (n - 1) 이다.
   - 이렇게, 최대값 문제의 상한과 하한은 같으며, 이는 풀리지 않은 문제가 없다는 것을 의미한다.
      - 왜냐하면, 비교가 하한만큼 필요한 경우가 있고, 상한의 비교로 항상 문제를 풀 수 있기 때문이다.
      - 또한, 풀리지 않은 문제가 없다는 것을 다르게 말하면, 해당 문제가 완전히 풀렸음이 된다.
- 토너먼트 선택에 대해 공부했다.
   - 차례대로 2개의 원소를 짝지어 비교하고, 둘 중에 큰 값을 선택하는 과정을 반복하는 방식이다.
   - 각 과정을 회차(round) 라고 하며, 회차마다 선택된 값들은 다음 회차로 진출하게 된다.
   - 만약, 짝이 지어지지 않아 하나의 값이 남는 상황이 생기면, 해당 값은 부전승으로 취급한다.
   - 마지막 회차인 결승에서 선택된 값이 최대값이 되며, 이 방법도 (n - 1) 번의 비교가 필요하다.
- 최대값, 최소값 찾기 문제에 대해 공부했다.
   - 선택 문제에서 k = 1, n 인 경우를 최소값, 최대값 찾기 문제라고 한다.
      - 말 그대로, 최대값과 최소값을 동시에 찾아야 하는 그런 문제다.
      - 이 때, 최소값은 최대값을 구하는 방법을 반대로 적용하여 구할 수 있다.
   - 왼쪽부터 차례대로 모든 항목을 하나씩 비교하는 방법을 이용하면 (2n - 3) 번의 비교가 필요하다.
      - 왜냐하면, 필요한 비교 횟수는 최대값 찾기 (n - 1), 나머지에서 최소값 찾기 (n - 2) 이기 때문이다.
   - 이 때, 토너먼트 방식을 적용하면 상한인 (2n - 3) 의 비교 횟수를 더 적게 만들 수 있다.
      - 최소값은 최대값 찾기 토너먼트에서 가장 먼저 탈락한 값 중에 반드시 존재한다.
      - 이렇게 가장 먼저 탈락한 값의 개수는 입력의 크기인 n의 절반, 즉 (n / 2) 다.
      - 따라서, 최대값을 구할 때는 (n - 1), 최소값을 구할 때는 (n / 2) - 1 번의 비교만 수행하면 된다.
      - 이 둘을 합하면 ((3 / 2) * n) - 2 가 되며, 이는 (2n -3) 보다 작은 값이다.
      - 이렇게, 어떤 경우에도 문제를 풀 수 있는, 더 작은 연산 횟수인 상한을 구했다.
   - 수업에서는 하한에 대해 따로 증명하지 않았지만, 구하는 과정에서 이미 증명되었다고 할 수 있다.
   - 이 때, 상한과 하한 모두 ((3 / 2) * n) - 2 로 같기 때문에, 이 문제도 완전히 풀렸다고 할 수 있다.
- 가장 작은 값과 두 번째로 작은 값 찾기 문제에 대해 공부했다.
   - 선택 문제에서 k = 1, 2 인 경우이며, 이 문제 또한 (2n - 3) 번의 비교로 해결할 수 있다.
   - 최대값, 최소값 찾기 문제처럼, 토너먼트 선택에서 얻은 정보로 전체 비교 횟수를 줄일 수 있다.
   - 비교하는 2개의 값 중에 더 작은 값을 선택하는 토너먼트를 진행하여, 최소값을 구할 수 있다.
   - 이 때, 두 번째로 작은 값은 가장 작은 값을 만나기 전까지는 무조건 작은 값으로 선택된다.
   - 하지만, 두 번째로 작은 값이 가장 작은 값과 만나는 시점이 마지막 회차인 결승이 아닐 수도 있다.
      - 만약, 두 번째로 작은 값이 1회차에서 가장 작은 값과 만난다면, 그대로 탈락하게 된다.
   - 따라서, "두 번째로 작은 값은 '가장 작은 값과 함께 비교되어 탈락한 값' 중에 있다" 고 할 수 있다.
      - 이 때, 가장 작은 값과 함께 비교되어 탈락한 값 중에서 가장 작은 값은 두 번째로 작은 값이 된다.
   - 그리고, 이렇게 가장 작은 값과 함께 비교되어 탈락한 값의 개수는, 토너먼트의 회차 수와 같다.
      - '두 번째로 작은 값을 찾는 데에 회차 수에서 1을 뺀 수만큼의 비교 횟수가 필요하다' 고 할 수 있다.
      - 입력의 크기 n이 주어질 때, 토너먼트의 회차 수를 구하기 위해선 log2(n) 을 올림 해야 한다.
   - 최종적으로, 최소값 찾기에 필요한 비교 횟수와 두 번째로 작은 값 찾기에 필요한 비교 횟수를 합치면 된다.
      - $(n - 1) + ({\lceil}log2(n){\rceil} - 1) = n - {\lceil}log2(n){\rceil} - 2$ 다.
   - 따라서, 이 문제의 상한과 하한은 $n - {\lceil}log2(n){\rceil} - 2$ 라고 할 수 있다.
      - 문제를 푸는 과정에서 이미 상한과 하한이 증명되었기 때문이다.
