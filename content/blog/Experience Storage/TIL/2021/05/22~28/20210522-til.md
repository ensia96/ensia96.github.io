# 오늘 한 일

- 자료 구조, 알고리즘 수업 정리
   - '알고리즘 선택문제 Quick Select' 라는 강의 내용을 정리했다.
   - 강의 내용 정리는 마무리하지 못했다. `(내일은 꼭!)`
- 선택 문제에 대해 공부했다.
   - 선택 문제에서 k = (n / 2) 이면, 크기대로 정렬된 값의 중앙값(Median) 을 찾아야 한다.
   - 선택 문제에서는 'Quick Select' 와 'Median of Medians(MoM)' 라는 알고리즘이 중요하다.
- Quick Select 알고리즘에 대해 공부했다.
   - n개의 숫자가 들어있는 리스트 L이 있다고 가정한다.
   - 우선, L에서 기준이 될 숫자인 p(pivot) 을 선택한다.
      - 첫 번째 값(L[0]), 마지막 값(L[n - 1]), 또는 임의로 선택하면 된다.
   - p를 기준으로 L에 있는 값을 크기에 따라 분류한다.
      - p보다 작으면 A, 크면 B, 같으면 M이라는 집합에 값을 집어넣는다.
      - 이 때, 같은 집합에 들어있는 값들 사이의 대소 관계는 알 수 없다.
   - k번째로 작은 수가 어떤 집합에 포함되어 있는지 파악한다.
      - 이를 위해, 크기대로 집합을 나열하여, 각 집합의 원소 개수와 k를 비교한다.
      - k번째로 작은 수가 속한 집합이 확인되면, 알고리즘을 재귀적으로 호출한다.
      - 이 때, 집합이 아닌 '전체에서 k번째로 작은 수를 찾아야 한다' 는 점을 유의해야 한다.
      - 만약 k번째로 작은 수가 M에 포함되어 있다면, p의 값을 그대로 반환한다.
- Quick Select 알고리즘의 의사 코드를 작성해봤다.
   - 우선, 리스트 L과 자연수 값 k를 인자로 받는 함수를 정의한다.
   - p(pivot) 를 정하고, 집합을 대신할 리스트 A, B, M 을 초기화한다.
   - L에 있는 각각의 원소들을 p와 비교하도록 반복문을 추가한다.
   - p와 원소의 크기를 비교하기 위해 반복문 내부에 조건문을 추가한다.
      - p보다 작으면 A, 크면 B, 아니면 M에 삽입하도록 구성한다.
   - A의 원소 개수가 k보다 큰지를 비교하는 조건문(if) 을 추가한다.
      - 참이면, A에서 k번째로 작은 수를 찾도록 재귀적으로 호출한다. (+ 반환)
   - A와 M의 원소 개수가 k보다 작은지를 비교하는 조건문(elif) 을 추가한다.
      - 참이면, B에서 (k - A와 M의 원소 개수) 번째로 작은 수를 찾도록 재귀적으로 호출한다. (+ 반환)
   - if, elif 모두 아닌 경우를 처리하도록 조건문(else) 을 추가한다.
      - 이는 k번째로 작은 수가 M에 포함된다는 것을 의미하므로, p의 값을 반환한다.
