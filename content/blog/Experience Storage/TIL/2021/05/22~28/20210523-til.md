# 오늘 한 일

- 자료 구조, 알고리즘 수업 정리
   - '알고리즘 선택문제 Quick Select' 라는 강의 내용을 정리했다.
- Quick Select 알고리즘의 수행 시간에 대해 공부했다.
   - Quick Select 알고리즘은 n개의 숫자 중에서 k번째로 작은 수를 찾는 알고리즘이다.
   - 임의의 숫자를 pivot 으로 지정하여, pivot 값을 기준으로 다른 숫자들을 분류한다.
      - pivot 보다 작은 수(A), pivot 보다 큰 수(B), pivot 과 같은 수(M) 로 나누어진다.
   - 크기에 따라 분류된 집합 중, 찾아야 하는 값이 속한 집합을 파악하여 문제를 푼다.
   - 각 집합의 원소 개수와 k를 비교하여, 찾고자 하는 값이 속한 집합을 확인할 수 있다.
   - 찾아야 하는 수가 pivot 과 같으면 pivot 을 반환하고, 아니면 재귀적으로 해결한다.
   - 재귀적으로 해결할 때, 찾아야 하는 수가 속한 집합의 크기에 따라 효율이 달라진다.
   - pivot 을 제외한 모든 값이 A나 B 중에 하나의 집합에 모이는 것이 최악의 경우이다.
      - 최악의 경우에는 재귀적으로 호출될 때마다, 비교 대상에서 pivot 하나만 제외된다.
      - 따라서, 문제 해결에 필요한 비교 횟수는 n + (n - 1) + (n - 2) + ... 이 된다.
      - 점화식으로 표현하면 T(n) = T(n - 1) + n, 이를 전개하여 수행 시간을 알 수 있다.
      - 점화식은 1 + 2 + … + n = (n * (n + 1)) / 2 이 되고, 수행 시간은 O(n^2) 이 된다.
      - 하지만, 이렇게 극단적으로 좋지 않은 상황이 발생하는 경우는 드물다.
   - pivot 을 제외한 모든 값이 A와 B에 정확히 반으로 나누어지는 것이 최선의 경우이다.
      - 최선의 경우에는 재귀적으로 호출될 때마다, 비교 범위가 절반으로 줄어든다.
      - 따라서, 문제 해결에 필요한 비교 횟수는 n + (n / 2) + (n / 4) + ... 이 된다.
      - 점화식으로 표현하면 T(n) = T(n / 2) + n, 이를 전개하여 수행 시간을 알 수 있다.
      - 점화식은 n * (1 + (1 / 2) + (1 / (2^2)) + ... + (1 / (2^k))), 즉, O(n) 이 된다.
   - 이렇게, 최선의 경우에는 O(n), 최악의 경우에는 O(n^2) 으로, 엄청나게 차이가 난다.
   - 하지만, 극단적이지 않은 일반적인 상황, 즉, 평균의 경우, 수행 시간은 O(n) 이다.
      - 평균의 경우에 대한 증명은 강의 노트에 있는 내용을 살펴봤다. `(쉿, 비밀)`
   - 따라서, 일반적인 상황에서도 최선의 경우와 비슷한 수행 시간을 가진다고 할 수 있다.
   - 이는 Quick Select 알고리즘이 상당히 빠른 알고리즘이라는 것을 의미한다.
