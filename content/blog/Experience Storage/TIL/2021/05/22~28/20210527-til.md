- Quick Select 알고리즘에 대해 공부했다.
   - pivot 을 선택하고, 값을 구분한 다음, 재귀적으로 호출하거나 값을 바로 반환하는 알고리즘이다.
   - 이 때, pivot 에 의해 나뉘는 두 집합 A와 B의 원소 개수에 따라서 전체 수행 시간이 달라진다.
   - 최악의 경우에 필요한 비교 횟수는 O(n^2), 평균의 경우에 필요한 비교 횟수는 O(n) 이다.
   - 일반적인 상황의 기대값인 평균의 경우가 O(n) 이므로, 굉장히 좋은 알고리즘이라 할 수 있다.
   - 최악의 경우는 A와 B가 극단적으로 나뉘기 때문에 발생하므로, pivot 을 잘 선택할 필요가 있다.
   - 이 때, 잘 선택된 pivot 은 A와 B의 원소 개수를 (n / c) 개 이하로 만드는 pivot 이다.
   - 왜냐하면, 재귀적으로 호출될 때마다 비교해야 할 값의 개수가 줄어들 것이기 때문이다.
      - 비교 대상이 n개에서 (n / c) 개, (n / c) 개에서 (n / c^2) 개, ... 처럼 점점 줄어든다.
   - 이러한 과정은 (n / c^k) 의 값이 1이 될 때까지 반복되며, 총 반복 횟수는 logc(n) 이다.
   - 이 때, 알고리즘은 매번 pivot 을 고를 때(p), 다른 수와 비교할 때(n) 비교를 수행한다.
   - 따라서, 문제를 해결하기까지 필요한 비교 횟수는 (p + n) * logc(n) 이라고 할 수 있다.
   - (p + n) * logc(n) 는 (p + n) * log2(n), (p + n) * log2(n) 는 2n * log2(n) 로 표현할 수 있다.
   - 이 때, 2n * log2(n) 을 빅오 표기법으로 표기하면 O(n log n) 으로, O(n^2) 보다 효율적이다.
   - 이렇게, pivot 을 잘 선택하면, 전체 비교 횟수를 O(n^2) 에서 O(n log n) 으로 줄일 수 있다.
   - 다시 말해, A와 B의 원소 개수를 계속 (n / c) 보다 작게 만들면, 무조건 O(n log n) 이 보장된다.
   - 따라서, 어떠한 상황에서라도 문제를 더 효율적으로 해결하기 위해서는 pivot 을 잘 선택해야 한다.
