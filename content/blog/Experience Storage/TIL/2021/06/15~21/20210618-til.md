# 오늘 한 일

- Crash Course 내용 정리
   - '7. 도메인 이름 체계' 까지 정리했다.
- 전송 제어 프로토콜에 대해 공부했다.
   - 모든 정보가 정확하게 전달해야 할 땐, 전송 제어 프로토콜(TCP) 을 사용한다.
   - UDP와 마찬가지로, TCP도 IP 패킷의 정보 페이로드 내부에 정보를 저장한다.
   - TCP는 일반적으로 IP와 함께 사용되어서, 이러한 조합을 TCP/IP 라고 부른다.
   - TCP는 UDP처럼 자체적인 헤더를 가지며, 그 헤더에 포트와 체크섬을 저장한다.
- 일련번호와 승인 신호에 대해 공부했다.
   - TCP는 나열되는 순서에 따라, 각각의 패킷에 연속적으로 일련번호를 부여한다.
   - 수신 컴퓨터는 이러한 일련번호를 이용해, 전달받은 패킷들을 정렬할 수 있다.
   - 또, TCP는 패킷의 수신을 완료했다는 것을 알리는 승인 신호의 사용을 정의한다.
   - 수신 컴퓨터의 경우, 패킷을 받거나, 체크섬이 일치할 때 승인 신호를 보낸다.
   - 송신 컴퓨터는 이러한 승인 신호를 받고 난 후에, 다음 패킷을 전송할 수 있다.
   - 송신 컴퓨터는, 일정 시간 동안 승인 신호를 받지 못하면, 패킷을 재전송한다.
   - 이 때, 수신 컴퓨터의 승인 신호 전송이 누락되거나, 지연을 겪을 수도 있다.
   - 패킷을 하나가 아닌, 여러 개를 보냄으로써, 대역폭을 크게 늘릴 수도 있다.
      - 왜냐하면, 패킷마다 받아야 하는 승인 신호를 기다리지 않기 때문이다.
   - 전송/승인 관련 정보를 활용해, 네트워크가 정체되는 정도를 추론할 수 있다.
      - 이렇게 얻은 정보로 패킷 전송량을 조절하여 정체 제어를 할 수 있다.
- TCP의 특징에 대해 공부했다.
   - TCP는 기본적으로 비순차적인 패킷 전달과 누락된 패킷을 처리할 수 있다.
   - 하지만, 승인 신호 패킷이 통신 수를 두 배로 늘린다는 것은 큰 단점이다.
   - 정보의 정확성보다 빠른 통신이 중요한 경우에 적합하지 않다고 할 수 있다.
- 도메인 이름 체계에 대해 공부했다.
   - 컴퓨터를 웹 사이트에 연결하기 위해서는, IP 주소와 포트가 필요하다.
   - IP 주소와 포트를 웹 브라우저 주소창에 입력하면 웹 사이트로 연결된다.
   - 이 때, IP 주소를 기억하기 쉬운 도메인 이름에 연결하는 서비스가 있다.
   - 이러한 네트워크 관련 서비스를 도메인 이름 체계(DNS) 라고 부른다.
   - 주소창에 문자 주소를 입력하면, 웹 브라우저는 DNS에 주소 조회를 요청한다.
   - 일반적으로 ISP에서 제공되는 DNS는 등록소를 참조해 IP 주소를 확인한다.
   - 주소가 있으면 해당 주소로 연결되지만, 없으면 DNS 실패 오류가 발생한다.
