- 자료 구조, 알고리즘 수업 정리
   - '알고리즘 - 분할정복 - 큰수 곱셈 (Karatsuba 알고리즘)' 라는 강의 내용을 정리했다.
- 카라추바 알고리즘의 정복 아이디어에 대해 공부했다.
   - 문제를 단순히 나누기만 했을 때, 카라추바 알고리즘이 필요로 하는 수행 시간은 O(n^2) 이다.
   - 그리고, 원래 문제를 해결하는 데 필요한 작은 문제들의 개수에 따라서, 수행 시간이 달라진다.
   - 카라추바는, 재귀적으로 풀어야 하는 문제의 수를 줄여서, 알고리즘의 수행 시간을 줄이고자 했다.
   - 이 때, 입력 u와 v에 대한 등식은 u * v = (xw * 10^n) + ((xz + yw) * 10^(n / 2)) + yz 이다.
   - 카라추바의 아이디어는, 등식의 xz + yw 부분을 (x + y)(w + z) - xw - yz 로 바꾸는 것이었다.
   - 이렇게 식을 변형하면, xz와 yw를 각각 계산하지 않아도 되며, 한번에 xz + yw 를 구할 수 있다.
   - 따라서, 원래 문제를 해결하기 위해 풀어야 하는 절반 크기의 문제는, 1개 줄어들어, 3개가 된다.
   - 이 때, (x + y)(w + z) - xw - yz 를 계산하는 데 필요한 기본 연산 횟수도, n에 비례하는 횟수다.
   - 그러므로, 알고리즘을 수행하는 데 필요한 기본 연산의 횟수는, n에 비례하는 상수 횟수 cn이다.
   - 이렇게 구성된 점화식은 T(n) = (3 * T(n / 2)) + cn 이며, 이를 풀면, O(n^log2(3)) 이 된다.
   - 2 = log2(4) 이며, log2(3) < log2(4) 이므로, O(n^2) < O(n^log2(3)) 이라고 할 수 있다.
