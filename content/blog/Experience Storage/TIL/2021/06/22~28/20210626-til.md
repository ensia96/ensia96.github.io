# 오늘 한 일

- Crash Course 내용 정리
   - '3. HTTP' 의 초반부까지 정리했다.
- 하이퍼링크에 대해 공부했다.
   - 월드 와이드 웹의 기본 구성 요소는 내용이 있는 하나의 문서이며, 이를 페이지라고 한다.
   - 이 때, 클릭하면 다른 페이지로 이동하게 되는 문자나 이미지를 하이퍼링크라고 부른다.
   - 이러한 하이퍼링크를 통해, 여러 가지 정보들이 서로 연결되면서, 거대한 망이 형성된다.
   - 버니바 부시가 1945년에 발표한 기사에는, 하이퍼링크라는 개념이 자세히 묘사되어 있다.
   - 이러한 하이퍼링크를 포함하는 텍스트인 하이퍼텍스트를 사용하여 문서를 작성할 수 있다.
   - 이렇게 작성된 문서를 웹 페이지라고 하며, 이는 가장 흔한 유형의 하이퍼텍스트 문서다.
- URL에 대해 공부했다.
   - 웹 페이지가 하이퍼링크를 통해 서로 연결되려면, URL이라는 고유한 주소가 있어야 한다.
   - 우선, 사용자가 어떤 사이트를 요청하면, 컴퓨터는 맨 처음에 DNS 조회 작업을 처리한다.
   - DNS 서버는 입력받은 도메인 이름을 조회하여, 도메인 이름에 대한 IP 주소를 응답한다.
   - IP 주소를 전달받은 웹 브라우저는 해당 IP 주소를 가진 컴퓨터로 TCP 연결을 설정한다.
   - 이 때, 연결 대상은, 웹 서버라고 불리는 특별한 소프트웨어를 실행하고 있는 컴퓨터가 된다.
   - 이렇게, 도메인 이름의 웹 서버에 연결한 다음에는, 하이퍼텍스트 페이지를 요청해야 한다.
