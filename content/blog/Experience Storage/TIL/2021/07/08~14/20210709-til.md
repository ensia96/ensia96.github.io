## 오늘 한 일

오늘은 사용자 관련 기능에 대한 테스트 코드 작성을 마무리했고, 채팅 기능을 조사했다.

- 테스트 코드는 당장 가능한 부분들만 간단하게 리팩토링을 진행했고, 실행까지 해봤다.
- 채팅 기능을 구현한 후에, 동작 확인까지 했고, 나중에 필요할 것 같은 자료들도 모았다.

<br>

오늘은 Django 기반으로 비동기 채팅 기능을 구현해봤고, Redis 를 처음으로 사용해봤다.

- JavaScript 에서만 보던 `async` / `await` 키워드를 Python 에서 보니 반가웠다. ㅎ
- 그리고, 처음 사용해보는 PostgreSQL 에도 접속해봤고, 간단한 쿼리도 직접 날려봤다.
- 블로그 관련 작업이 끝나면, 오늘 배운 것들을 모두 짧은 글 시리즈로 남겨둬야겠다.

## 생각 정리

취업에 중요한 것은 프레임워크, 라이브러리처럼 실제로 사용하는 기술이라는 생각이 들었다.

- 프레임워크를 오래 사용한 사람과 그렇지 않은 사람 사이에서 생기는 차이는 큰 것 같다.
- 블로그 관련 작업까지 모두 끝낸 후에는, 공부하고 글로 작성하는 범위를 더 넓혀봐야겠다.
- 이론은 이론대로 공부하고, 새로운 무언가도 직접 만들어보면서 성장하는 것이 좋을 것 같다.
- 그리고, 글이나 공부의 주제는 확실히 어떤 작업을 하는 과정에서 많이 얻게 되는 것 같다.
- 아까운 시간을 낭비하지 않고 의미 있게 보내기 위해서는, 새로운 도전이 중요한 것 같다.

<br>

어젯밤에는, 백엔드 동료분과 'Google Meet' 으로 이야기하면서, 경험의 중요성에 대해 배웠다. 

- 백엔드의 경우, 언어나 프레임워크 외에, 운영에 관련된 부분들도 중요하다는 것을 배웠다.
- 처음 들어본 기술들이 엄청나게 많아서, 이름들을 물어보고, 메모장에 전부 적어두었다.
- 개발 직군을 바꾸지 않는 이상, 모두 공부해야 한다고 생각하니, 조금 어질어질했다.. ㅋㅋ
