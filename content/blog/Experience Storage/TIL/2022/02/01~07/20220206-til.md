오늘 알고리즘 문제는 '2차원 평면 위의 정점들을 연결하는 최소 신장 트리' 를 구해야 하는 문제였다.

- 우주신과의 교감 문제는 정점 사이의 거리를 미리 구해둔 후, 간선 정보로 활용하는 식으로 풀었다.
- 이미 연결된 두 정점이 있다는 조건 때문에 조금 헤매였다.. (처음 보는 조건이라서 많이 당황했음;)  
  `(이미 연결된 두 정점 사이의 거리를 0으로 두면, 추가할 간선의 비용만 합산됨 <- 좀 늦게 깨달았음..)`

<br>

'두들낙서' 님의 강의 영상을 보고 공부했다. (dynamic\_cast 와 RTTI.. 동작 원리는 대충 이해했는데..)

- 가상 함수를 포함하는 클래스를 다형 클래스라고 부른다는 것을 알게 됐다. (다형성은 더 나중에..)
- 다형 클래스의 객체에는 자신의 자료형에 대한 정보를 가리키는 포인터가 저장된다는 것을 배웠다.  
  `(임의의 다형 클래스 객체를 메모리상에서 봤을 때, 이 포인터는, 메모리 공간의 가장 앞부분에 저장됨.)`
- C++ 에는 실행 시간에 자료형에 대한 정보를 알 수 있게 해주는 'RTTI' 기능이 있다는 것을 배웠다.
- RTTI 기능은, 가상 함수로 인해 일어나는 동적 바인딩에서도 활용되는 기능이라는 것을 알게 됐다.  
  `(오버라이딩된 가상 함수에 대한 동적 바인딩이 일어날 때도 실행 시간에 자료형을 확인할 수 있어야 함.)`
- dynamic\_cast 를 통해 어떤 자료의 자료형을 실행 시간에 동적으로 변환할 수 있다는 것을 배웠다.  
  `(dynamic_cast<Type>(Target) 형태로 작성 -> Target(대상) 이 Type(자료형) 으로 변환됨.)`
- 다형 클래스가 아닌 클래스에 대해서는, dynamic\_cast 로 다운캐스팅을 할 수 없다는 것을 배웠다.  
  `(다형 클래스가 아닌 클래스는, 실행 시간에 자료형을 확인할 방법이 없기 때문 -> RTTI 기능 지원 x)`
- dynamic\_cast 는, 형 변환에 성공하면 자료값을, 실패하면 NULL 값을 반환한다는 것을 알게 됐다.
- dynamic\_cast 가, 성능 측면에서 봤을 때 그렇게 좋은 기능이 아니라는 것을 배웠다. (RTTI 때문..)  
  `(RTTI 기능에 관련된 여러 부수적인 작업들이 실행 시간에 처리되기 때문 -> 프로그램 성능이 떨어짐.)`
