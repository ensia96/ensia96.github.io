## 1. 2018 KAKAO BLIND RECRUITMENT(1차)

> 출처 : ['카카오 신입 공채 1차 코딩 테스트 문제 해설'](https://tech.kakao.com/2017/09/27/kakao-blind-recruitment-round-1/)

- 업무에서 있을 만한 상황을 가정, 다양한 분야에서 출제한 비교적 쉬운 수준의 독창적인 문제
- 자료 구조, 알고리즘 등의 전산학 기초를 충분히 학습한 사람은 누구나 풀 수 있을 만한 문제

### 1-1. 비밀 지도

> 비트 연산(Bitwise Operation)

> 정답률 : 81.78%

- 예시에 2진수 처리가 필요함을 암시하는 힌트가 있고, 둘 중 하나가 1인 경우를 처리하면 됨
- if else 구문을 이용해 푸는 방법도 있지만, 비트 연산 활용 능력을 확인하는 것이 문제의 의도

### 1-2. 다트 게임

> 문자열 처리(String Manipulation), 토큰화(Tokenizing) 와 의미 분석(Semantic Analysis)

> 정답률 : 73.47%

- 앞에서부터 한 글자씩 잘라서 처리하거나, 컴파일러처럼 토큰화와 의미 분석을 통해 계산 가능
- 정규식으로 비교적 쉽게 풀 수 있으며, 두 자리 숫자와 음수 점수가 있다는 점에 유의해야 함

### 1-3. 캐시

> LRU 캐시 교체 알고리즘 구현(검색 가능. 단, 출처 명시는 필수)

> 정답률 : 45.26%

- 입출력에 캐시 사이즈 0이 포함되어 있어서, 단순한 코드 복사/붙여넣기만으로는 해결 불가
- 입출력 예제 결과를 분석하면 쉽게 풀 수 있으며, 비정상적인 상황에 대한 예외 처리가 필요

### 1-4. 셔틀버스

> 문자열로 주어지는 시간을 조건에 맞게 계산

> 정답률 : 26.79%

- 일부 조건이 까다로운 편이며, 전체적인 계산은 어렵지 않지만, 정확히 계산하는 것이 어려움

### 1-5. 뉴스 클러스터링

> 자카드 유사도를 계산하는 프로그램 작성 (다중집합(multiset) or 병합 정렬(Merge Sort) 활용)

> 정답률 : 41.84%

- 기본적으로, 문제 풀이에 필요한 요소인 자카드 유사도와 다중집합에 대한 설명이 주어짐
- 일부 언어에서 기본적으로 제공하지 않고, 자주 접하는 자료 구조가 아닌 다중집합을 활용
- 다중집합 자료 구조를 사용하지 않아도, 정렬된 배열과 병합 정렬을 응용하는 풀이법도 있음
- 교집합과 합집합을 잘 구하고, 공집합 나눗셈(division by zero) 에 유의하면 쉽게 풀 수 있음

### 1-6. 프렌즈4블록

> 게임 요구 사항을 구현

> 정답률 : 48.01%

- 블록 매트릭스를 생성하여 스캔하고, 조건에 맞는 블록 덩어리를 제거하는 작업을 반복해야 함
- 제거할 블록이 남아있지 않을 때까지 작업을 반복한 후, 사라진 블록 자리의 수를 계산하면 됨

### 1-7. 추석 트래픽

> 초당 최대 처리량이 되는 구간 윈도우 찾기

> 정답률 : 17.99%

- 전부 스캔하기에는 범위가 너무 넓고, 단위 또한 작아서, 필요한 연산의 수가 너무 많아짐
- 타임아웃이 발생하는 것을 막기 위해, 스캔 범위를 줄일 방법을 도출해내는 것이 중요함
- 이러한 방법은, 각 로그의 시작과 끝에서 요청량이 변한다는 규칙을 통해 알아낼 수 있음
- 이 방법 외에도, 윈도우를 이용하지 않고, O(n log n) 의 시간 복잡도로 푸는 방법도 있음

## 2. 2018 KAKAO BLIND RECRUITMENT(2차)

> 출처 : ['카카오 신입 공채 2차 코딩 테스트 문제 해설'](https://tech.kakao.com/2017/10/24/kakao-blind-recruitment-round-2-2/)

- 1차에서는 자료 구조, 알고리즘 등 전산학 기초 지식과 기본적인 구현 역량을 확인
- 2차는 빠른 습득력, 요구사항 분석과 기능 설계 능력, 모니터링/개선 능력을 확인

> 문제 내용 : 이미지 특징값을 수집하는 Crawler 구현

### 1. REST API와 JSON Parsing

- 우선, 토큰을 발급받은 후에, 테스트 전용 서버로 연결되는 5개 카테고리의 URL 을 받아야 함
- 카테고리의 문서를 읽고, 그 안에 있는 이미지들의 특징값을 추출하여 저장/삭제하는 문제임

### 2.  반복적인 모니터링과 최적화(제한된 토큰 유효시간)

- 토큰 유효기간은 10분으로 설정, 점수 현황판을 공개하여 지원자의 최적화 작업을 유도함
- 문제가 발생할 때마다 로그를 남기고, 모니터링을 통해 원인이나 병목 현상을 찾도록 유도함

### 3. 평가 메트릭 분석 후에 전략 수립

- 누락되거나 삭제되지 않은 이미지, 잘못된 정보의 수에는 패널티가 붙도록 기준을 정함
- 실제 상황에서 얼마나 큰 문제가 될 수 있는지에 따라서 패널티의 크기를 다르게 적용함
- 참가자들이 불필요한 요청의 수를 줄이도록, 전체 쿼리의 양에도 약간의 패널티를 부여함

### 4. 고득점을 위한 병렬 처리

- 가장 자주 호출하게 되는 이미지 엔드포인트의 처리 속도를 조절하여 병렬 처리를 유도함
- 처리 속도는 16ms ~ 120ms 이므로, 네트워크 지연을 포함해 초당 요청 제한은 50건 정도
- 이 때, 테스트 전용 서버의 처리 속도는 배치 처리 개수에 따라서 exp 그래프 형태를 띰
- 따라서, 최대 배치 처리 사이즈인 50개보다, 30 ~ 40개 정보의 요청에서 가장 효율이 좋음

### 5. 예외 처리

- 크롤링 작업 중에 사용자가 이미지를 삭제하거나 서버가 다운되는 경우도 반영되어 있음
- 따라서, 확률적으로 성공하는지, 무조건 실패하는지에 따라 재시도나 블록 처리가 필요함

### 6. 다양한 시나리오에 대응

- 중복 추가, 추가 후 바로 삭제, 없는 이미지 삭제, 추가 삭제 추가가 반복되는 상황도 있음
- 이 때, 모든 이미지를 색인하면서 발생하는 공간 낭비는 간단한 메모리 캐시로도 해결 가능
- 연산 목록은 단순히 set 자료형을 활용해 중복을 상쇄하면 문제가 발생하도록 구성되어 있음
- 또한, 추가/삭제 연산이 발생하는 범위가 넓어서, 시간과 정확성 사이에서 의사결정이 필요

### 7. 카테고리별 특성 파악

- 카테고리마다 추가/삭제 비율과 URL 갱신 시간에 차이가 있어서, 적절한 전략이 필요함
- 카테고리별 특징을 확인하고, 정보 수집량과 패널티 사이에서 의사결정을 내릴 필요가 있음
- 어떤 카테고리를 대상으로 할지, 몇 개의 카테고리를 대상으로 할지 등을 결정해야 함

### 합격선과 출제 의도

- 병렬 처리를 하지 않고, 예외 처리와 배치 처리만 해도 받을 수 있는 점수인 8만 점이 합격선
- 실무에서도 흔히 발생하는 트레이드오프(의사결정 상황) 에 대한 대처 능력을 보는 것이 의도
- 평균 점수는 예외 처리만 하면 5만, 배치 처리까지 하면 14만, 병렬 처리까지 하면 25만 점임

## 3. 2018 KAKAO BLIND RECRUITMENT(3차)

> 출처 : ['카카오 신입 공채 3차 코딩 테스트 문제 해설'](https://tech.kakao.com/2017/11/14/kakao-blind-recruitment-round-3/)

- 3차는 오프라인으로 진행되었기 때문에, 개발 환경이 다르고, 검색하는 것도 불가능함
- 또한, 코딩 테스트 외에도 컴퓨팅 관련 기본 지식을 확인하는 필기시험도 진행되었음
- 이 때, 코딩 테스트는 5문제 중 3문제, 필기시험은 40점을 넘기는 것이 합격 기준임
- 코딩 테스트는 1차와 난이도가 비슷하지만, 복합적으로 생각해야 하는 문제들이 출제됨

### 3-1. N진수 게임

> 반복문과 진법 변환을 이용해 순서대로 나열된 숫자들의 특정 값만 구하기(챔퍼나운 수 활용)

> 정답률 : 91.85%

- 진법 변환은 프로그래밍 언어를 처음 배울 때 연습 문제로 많이 풀어보기 때문에 설명 없음

### 3-2. 압축

> 문자열 정보를 압축하여 색인 배열로 출력하는 LZW 알고리즘 구현

> 정답률 : 95.80%

- 압축 알고리즘이라 어렵게 생각할 수도 있지만, 설명의 의사 코드(Pseudocode) 만 구현하면 됨
- 문자열과 배열을 다루는 능력이 기초적인 수준이어도 풀 수 있으며, 정답률이 가장 높은 문제임
