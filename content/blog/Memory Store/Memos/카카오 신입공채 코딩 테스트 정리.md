# 1. 2018 KAKAO BLIND RECRUITMENT(1차)

['카카오 신입 공채 1차 코딩 테스트 문제 해설'](https://tech.kakao.com/2017/09/27/kakao-blind-recruitment-round-1/)

## 1-1. 비밀 지도

> 비트 연산을 묻는 문제

- if else 구문이 아닌 or 연산자를 이용

## 1-2. 다트 게임

> 문자열 처리를 묻는 문제

- 앞에서부터 한 글자씩 잘라서 처리
- 간단한 컴파일러를 만들듯, 토큰화와 의미 분석을 통해 계산

## 1-3. 캐시

> LRU 캐시 교체 알고리즘을 구현하는 문제

- 캐시 사이즈 0이 포함되어 있어서, 단순한 복붙은 해결 불가
- 입출력 예제의 결과를 분석하면 난이도가 줄어듬

## 1-4. 셔틀버스

> 시간을 계산하는 문제

- 전체 계산은 어렵지 않지만, 정확하게 계산하는 것이 어려움

## 1-5. 뉴스 클러스터링

> 자카드 유사도를 설명하고, 자카드 유사도를 직접 계산하는 프로그램을 작성하는 문제

- 원소의 중복을 허용하는 다중 집합에 대해 설명함
- 자주 접하는 자료 구조가 아니며, 일부 언어에서는 기본적으로 제공하지 않음
- 다중집합 자료 구조를 사용하지 않아도, 원소를 정렬된 배열에 넣은 후에 병합 정렬을 응용할 수도 있음
- 다중 집합의 교집합, 합집합만 잘 구해내면 어렵지 않게 풀 수 있음

## 1-6. 프렌즈4블록

> 게임 요구 사항을 구현해보는 문제

- 블록 매트릭스를 생성하여 스캔하고, 조건에 맞는 블록 덩어리를 제거하는 작업을 반복해야함
- 이후, 더 이상 제거되지 않을 때 사라진 블록 자리의 수를 계산하면 됨

## 1-7. 추석 트래픽

> 초당 최대 처리량이 되는 구간 윈도우를 찾는 문제

- 처음부터 끝까지 스캔하기에는 범위가 너무 크기 때문에, 필요한 연산이 너무 많아짐
- 타임아웃을 해결할 수 있도록 스캔 범위를 줄이는 아이디어를 도출할 필요가 있음
- 이를 위해, 요청량이 변하는 순간이 각 로그의 시작과 끝이라는 규칙을 알아내야함
- 윈도우를 이용하지 않고도, O(n log n) 의 시간 복잡도로 풀 수 있는 방법도 있음
