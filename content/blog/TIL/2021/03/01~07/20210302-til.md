# 오늘 한 일

- Crash Course 내용 정리
   - '4' 의 내용 정리를 마무리했다.
   - 글을 다듬고 있는데, 고치고 싶은 부분이 너무 많다..
- 컴퓨터가 문자 정보를 처리하는 방법에 대해 공부했다.
   - 2진법을 활용하여 문자를 표현한 사례가 1600년대에도 있었다.(프랜시스 베이컨)
   - 당시에는 5비트를 이용해 알파벳에 대한 표기만 가능했다고 한다.
- ASCII 코드에 대해 공부했다.
   - 각각의 문자를 숫자로 변환해 7비트 단위로 표기하는 방식이다.
   - 컴퓨터에서 사용할 수 있는 특수 명령 코드들이 있다.(줄 바꿈 표기 문자 등)
   - 1바이트 단위에서는 여분 코드로 특수한 문자들을 인코딩할 수 있었다.
      - 덕분에, 다른 나라들은 각 국가가 사용하는 문자를 인코딩할 수 있었다고 한다.
      - 그 와중에 미국은 수학적 기호나 그래픽 요소 등을 추가했다고 한다.
   - 국가 별로 인코딩 방식이 달라서, 충돌 문제가 있었다.
   - 수천 개의 문자를 사용하는 국가에서는 이런 인코딩 체계를 사용할 수 없었다.
      - 문제를 해결하기 위해 다중 바이트(multi-byte) 인코딩이 개발되었다.
      - 이렇게 생겨난 인코딩 방식도 당연히 호환안됨 ㅋ
   - 인코딩 충돌 문제로 이상하게 표기된 문자는 'mojibake'(모지바케) 라고 표현한다.
      - 일본에서 붙인 표현인데, 당시 문제가 얼마나 심각했는지 검색하면 위키피디아에도 뜬다!
- 상호 운용성 개념에 대해 알게됐다.
   - 하나의 체계가 여러 다른 체계와 호환되는 성질이다.
- 유니코드에 대해 공부했다.
   - 아스키 코드의 문제를 해결하기 위해 고안되었다.
   - <a href='https://d2.naver.com/helloworld/19187' target='-blank'>
     'Naver-D2 의 포스팅'
     </a>
     을 보고, 컴퓨터에서 한글이 표현되기까지의 역사를 알게됐다.
- 아스키 코드의 인코딩 방식이 파일에도 적용되었다는 것을 배웠다.
   - 전에 봤던 파일 압축과 관련된 글이 떠올랐다.
   - 결국, 모두 0과 1로 표현된다니.. 매트릭스 영화가 보고싶어졌다.

# 생각 정리

- 어제부터 로파이 힙합을 듣기 시작했는데, 되게 괜찮은 것 같다.
   - <a href='https://www.youtube.com/channel/UCSJ4gkVC6NrvII8umztf0Ow' target='-blank'>'ChilledCow'</a>
     라는 유튜브 채널의 실시간 스트림에서 들을 수 있다.
   - 영상 속 캐릭터도 뭔가를 하고 있어서, 도서관에 온 기분을 느낄 수 있다.
   - 뭔가 차분하게 집중이 잘 되는 기분..
- 글을 정리하는 방법이 점점 발전하고 있다는게 느껴진다.
   - 처음에는 그냥 쭉 적다가, 중간중간 고치는 식이였고, 체계가 전혀 없었다.
   - 글을 여러 번 정리하면서, 조금씩 체계가 생겼다.
      - 현재는 전체 내용을 적어두고, 큰 구성을 나누고, 다듬는 등의 순서로 진행한다.
   - 글을 정리하는 속도나 완성도가 늘었다는게 확실히 느껴진다.
   - 글 다듬기에 쓰는 시간이 줄어든 만큼, 모르던 부분을 검색하는 시간이 늘었다.
   - 더 좋은 방법을 찾기 위해 노력하면, 앞으로도 많이 성장할 수 있을 것 같다.
