# 오늘 한 일

- Crash Course 내용 정리
   - '7. 논리 장치' 까지 정리했다.
- 전가산기의 회로 구성과 추상화에 대해 공부했다.
   - 올림 값까지 3가지 입력에 대해 계산할 수 있어야 한다.
   - 'A + B + C = (A + B) + C' 이므로, 계산을 2가지 과정으로 분산한다.
   - 가능한 최대 입력은 '1 + 1 + 1' 이므로, 2번의 계산 중 하나에서만 올림 값이 출력된다.
   - 첫 번째 계산을 처리하기 위해 반가산기를 추가한다.
   - 첫 번째 계산의 합과 3번째 입력값인 올림 값(C) 의 계산을 처리하기 위해 반가산기를 추가한다.
   - 가장 마지막에 추가한 반가산기에서 출력된 합을 최종 합으로 출력한다.(해당 자리수의 결과)
   - 2번의 계산 중 하나라도 올림 값을 반환한다면, 올림 값을 출력해야 한다.
   - 따라서, 2개의 반가산기에서 출력된 올림 값들에 대해 논리합 연산을 수행하면 된다.
   - 최종적으로 반환된 2개의 올림 값 출력을 입력받는 논리합 연산 회로를 추가한다.
   - 논리합 연산 회로의 출력을 최종 올림 값으로 출력한다.(해당 자리수 연산의 올림 값)
   - 이 과정을 단순화하면, 3개의 입력을 받아 2개의 결과를 출력한다고 할 수 있다.
- 리플 자리올림수 가산기에 대해 공부했다.
   - 임의의 수 'n' 이 있을 때, n자리의 2진수 2개를 더하는 연산을 처리한다.
   - 각 자리수의 비트들을 순서대로 계산한다.
   - 여러 자리수의 비트 계산을 처리하는데, 이를 '다중 비트(multi-bit) 연산을 처리한다' 고 표현한다.
   - 올림 수가 없는 첫 번째 비트들을 계산하는 1개의 반가산기가 있다.
   - 처음 이후의 계산들을 처리하는 'n - 1' 개의 전가산기가 있다.
   - 올림 값이 다음 가산기로 전달되는 형태가 물결(ripple) 모양과 비슷해서 이름이 붙여졌다.
   - 최종적으로 n개의 합과 1개의 올림을 출력한다.
- 오버플로에 대해 공부했다.
   - 사용하는 메모리 단위를 넘어서는 값이 생기는 현상이다.
   - 리플 자리올림수 가산기에서 '올림' 값이 출력되면 발생한다.
   - 수 많은 오류와 결함을 발생시킨다.
   - <a href='https://namu.wiki/w/%EC%98%A4%EB%B2%84%ED%94%8C%EB%A1%9C' target='-blank'>'나무위키'</a>
     를 보니, 맨 앞자리 비트가 부호를 표현하면 '-' 값으로 바뀐다고 한다.
- 자리올림수 예측 가산기에 대해 공부했다.
   - 오버플로 문제를 해결하기 위해 가산기를 추가하면, 처리 속도가 느려진다.
   - 속도 문제를 해결하는 방법으로, 올림 수에 대한 연산만 처리하는 별도의 회로를 구성한다.
   - 연산 처리 속도가 훨씬 더 빠르고, 현대 컴퓨터에서 많이 사용된다.
      - <a href='https://www.google.com/search?q=is+carry+look+ahead+adder+used+in+modern+computer&oq=is+carry+look+ahead+adder+used+in+modern+computer&aqs=chrome..69i57j33i21l2.3498j0j7&sourceid=chrome&ie=UTF-8' target='-blank'>'구글 검색 결과'</a> 를 참고
- ALU의 기본 산술 연산에 대해 공부했다.
   - 덧셈, 뺄셈, 부정, 증가, 감소, 통과 연산을 처리한다.
   - 저가의 프로세서에서도 모두 지원된다.
   - 고가의 프로세서의 경우, 곱셈 전용 회로를 포함하기도 한다.  
     `(자본주의의 향기를 느낄 수 있었다..)`
- 값이 0인지 판별하는 간단한 회로를 구성해봤다.
   - 하나의 값이 나올 때까지 OR 회로를 중첩 연결한다.
   - 하나의 입력이라도 1일 경우 1(참)이 되고, 모두 0인 경우 0(거짓)이 된다.
   - 출력의 진리값이 원하는 결과의 반대이므로, NOT 회로를 추가한다.
