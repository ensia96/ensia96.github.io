# 오늘 한 일

- Crash Course 내용 정리
   - '7. 가변 길이 명령어' 의 초반부까지 정리했다.
- 점프 명령어에 대해 알게됐다.
   - 메모리 주소 레지스터에 값을 덮어써서, 실행 위치를 변경한다.
   - 명령 순서를 변경하거나 일부 명령을 건너 뛸 수 있다.
   - 특정 조건에서만 수행되는 조건부 점프 명령어도 있다.  
     `ALU의 플래그를 활용해 조건 처리를 할 수 있다.`
- 정지 명령어는 프로그램을 종료하고 컴퓨터를 정지시킨다.
   - 정지 명령이 포함되지 않은 프로그램은 문제가 발생한다.
   - 프로그램을 구성하는 모든 명령을 수행하는 것에는 문제가 없다.
   - 마지막 명령 수행 후에도 다음 주소의 값을 수행하게 된다.
   - 해당 값에 해당하는 명령 코드가 없으면 문제가 발생한다.
- 점프 명령어를 활용해, 반복 동작을 수행하는 프로그램을 구성해봤다.
   - 레지스터에 값을 불러온 다음 시점으로 점프하도록 했다.
   - 종료 조건을 지정하지 않아서 무한 루프에 걸렸다.
- 무한 루프에 대해 공부했다.
   - 점프에 의해 정지 명령을 수행하지 못할 때 발생한다.
   - 의도적으로 무한 루프를 발생하는 경우도 있다.  
     `(어떻게 보면, 프로그램이 실행되고 있는 것도 무한 루프의 일종이다.)`
   - 종료 조건을 추가하면 무한 루프를 빠져나갈 수 있다.
- 조건부 점프 명령어를 활용하는 프로그램을 구성해봤다.
   - ALU의 'Negative' 플래그를 점프 조건으로 하는 명령어를 사용했다.
   - 임의의 값에서 다른 하나의 값을 빼는 연산을 반복해, 나누기 연산을 구현했다.
- 소프트웨어의 특징에 대해 공부했다.
   - 다양한 상황에서 동작할 수 있다.
   - 하드웨어의 한계를 극복할 수 있다.
   - 구성된 프로그램을 활용해 다른 프로그램을 구성할 수도 있다.
