# 오늘 한 일

- Crash Course 내용 정리
   - '5. 복잡성 비교' 까지 정리했다.
   - '12' 에 있던 오타를 수정했다. `(분명 지웠는데..;)`
- 알고리즘에 대한 내용을 정리했다.
   - 계산을 완료하기까지 수행되는 특정한 단계들이다.
   - 같은 결과를 출력해도, 더 우수한 알고리즘은 존재한다.
- 정렬에 대해 공부했다.
   - 컴퓨터 과학에서 가장 잘 알려진 문제다.
   - 컴퓨터는 항상 다양한 요소에 대한 정렬 작업을 수행한다.
   - 컴퓨터 과학자들이 수십 년 동안 발명해왔다.
   - 버블 정렬, 스파게티 정렬 등 다양한 정렬 알고리즘이 존재한다.
- 선택 정렬에 대해 공부했다.
   - 배열을 정렬하는 매우 기초적인 알고리즘이다.
   - 배열의 맨 처음부터 마지막까지, 모든 수를 비교한다.
   - N 개의 항목을 정렬하기 위해선 N * N 회의 반복이 필요하다.
   - 배열의 규모가 커질수록 반복 횟수는 심각하게 늘어난다.
- 복잡성과 빅오 표기법에 대해 공부했다.
   - 입력의 규모와 알고리즘 실행 단계 수 사이의 관계를 복잡성이라 한다.
   - 복잡성은 해당 알고리즘이 얼마나 빠른지, 느린지에 대한 근사치를 제공한다.
   - 이런 복잡성은 빅오 표기법으로 표현할 수 있다.
      - 선택 정렬 알고리즘의 복잡성은 O(N^2) 이라고 표현한다.
- 합병 정렬에 대해 공부했다.
   - 배열을 가장 작은 단위까지 쪼갠 후에, 합병하며 정렬하는 방법이다.
      1. 배열의 크기가 1이 될 때까지 나눈다.
      2. 맨 앞에 있는 2개의 배열부터, 각 항목의 크기를 비교해 새로운 배열에 넣는다.
   - 비교와 합병이 수행되는 횟수는 배열 항목 수에 비례하고, (N)  
     2개로 나뉜 배열들이 합병되는 단계의 수만큼 반복된다. (log N)
   - 복잡성은 O(NlogN) 이기 때문에, 선택 정렬보다 효율적이다.
