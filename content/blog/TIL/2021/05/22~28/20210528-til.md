# 오늘 한 일

- 자료 구조, 알고리즘 수업 정리
   - '알고리즘 선택문제 - MoM' 라는 강의 내용을 정리했다.
- MoM(Median of Medians) 알고리즘에 대해 공부했다.
   - 전체 n개의 숫자에서 정확히 절반이 아니더라도, 중간에 가까운 숫자를 찾는 알고리즘이다.
   - 우선, 숫자를 5개씩 모아서 그룹으로 만들고, 비교를 통해 각 그룹의 중앙값을 찾는다.
      - 이 때, (n / 5) 개의 그룹이 만들어지며, 상수 횟수(6) 의 비교를 통해 중앙값을 찾을 수 있다.
   - 다음으로, 중앙값들을 모아 medians 라고 한 후에, medians 에서도 중앙값(m\*) 을 찾는다.
      - 이 때, 중앙값의 개수 (n / 5) 개에서 중앙값을 찾기 위해, 자기 자신을 재귀적으로 호출한다.
   - MoM 알고리즘을 이용해, 선택 문제를 해결하는 데 필요한 비교 횟수를 T(n) 이라고 가정한다.
   - 그룹을 만들 때는 0번, 각 그룹의 중앙값을 찾을 때는 (n / 5) * 6 번의 비교가 필요하다.
   - 중앙값들의 중앙값을 찾기 위해 재귀적으로 호출했으므로, T(n) 에는 T(n / 5) 이 추가된다.
   - m\* 는 pivot 이 되며, m\* 보다 작은 값과 큰 값을 A와 B로 분류하기 위해 n번의 비교를 수행한다.
   - 선택 문제에서 찾는 k번째로 작은 숫자의 위치에 따라, 재귀적으로 호출하거나 바로 반환한다.
   - k번째로 작은 숫자가 A 또는 B에 있는 경우, A 또는 B에 대해 MoM 알고리즘을 재귀적으로 호출한다.
   - 이렇게 알고리즘을 재귀적으로 호출했으므로, T(n) 에는 T(|A|) 또는 T(|B|) 가 추가된다.
      - 이 때, MoM 알고리즘을 통해 선택된 pivot 은 A와 B의 원소 개수를 (n / c) 보다 작게 만든다.
      - 따라서, T(n) 에는 추가되는 T(|A|) 와 T(|B|) 를 T(n / c) 으로 바꿔서 추가할 수 있다.
   - 이렇게 파악한 비교 횟수를 모두 합하면 T(n) = T(n / c) + T(n / 5) + (11n / 5) 이 된다.
- MoM 알고리즘의 점화식에 대해 파악해봤다.
   - MoM 알고리즘의 비교 횟수 T(n) 은 T(n / c) + T(n / 5) + (11n / 5) 이라는 것이 확인되었다.
   - 여기서 중요한 것, 증명해야 하는 것은 ‘A와 B의 원소 개수가 (n / c) 이하라는 것’ 이다.
   - 그러기 위해선, 중앙값들의 중앙값을 선택했을 때에 c의 값이 얼마가 되는지 알아내야 한다.
   - 이를 위해, m\* 를 기준으로 각 그룹과 그룹에 있는 값들을 재배치하고, 구역을 나눠볼 수 있다.
   - m\* 를 기준으로 전체 그룹을 정렬하고, 각 그룹의 중앙값을 기준으로 내부의 항목들을 정렬한다.
   - 기준으로부터 왼쪽과 위쪽에는 기준보다 작은 값, 오른쪽과 아래쪽에는 큰 값이 위치하게 된다.
   - 왼쪽 위, 오른쪽 위, 왼쪽 아래, 오른쪽 아래로 구역을 나눠 각각 X, Y, Z, W 라고 이름을 붙인다.
   - 이 때, X는 m\* 보다 작은 값들을, W는 m\* 보다 큰 값들을 모두 포함하도록 구역을 설정한다.
   - X 구역과 W 구역에 포함되는 값의 개수는 각각, 전체 n에 대해서 (n / 4) 이상이라고 할 수 있다.
   - m\* 보다 작은 수를 포함하는 A에는 X가 무조건 포함되므로, A의 원소 개수의 최소값은 (n / 4) 이 된다.
   - A의 원소 개수의 최대값은 W를 제외한 모든 구역이 A에 포함될 때, 즉, n - (n / 4) = (3n / 4) 이 된다.
   - 따라서, A의 원소 개수는 최소 (n / 4) 최대 (3n / 4) 라고 할 수 있으며, 이는 B도 마찬가지다.
   - 그리고, MoM 알고리즘에 의해 선택된 pivot 은 A와 B의 원소 개수를 (n / c) 보다 작게 만든다.
   - 그러므로, A 또는 B의 원소 개수의 최대값인 (3n / 4) 을 (n / c) 과 같은 값이라고 할 수 있다.
   - 이를 통해 (3n / 4) = (n / c) 이고, (3 / 4) = (1 / c) 이므로, c = (4 / 3) 라는 것이 확인되었다.
   - 결국, T(n) = T(n / c) + T(n / 5) + (11n / 5) = T(3n / 4) + T(n / 5) + (11n / 5) 이 된다.
   - 이와 같은 복잡한 점화식을 전개하면, MoM 알고리즘이 수행하는 전체 비교 횟수를 파악할 수 있다.
